# 音声入力

## Background

gwt-tui にボイス入力機能を追加する。ホットキーでマイクキャプチャを開始し、whisper-rs（OpenAI Whisper のRustバインディング）でテキストに変換した後、PTY に送信する。macOS では AVFoundation、Linux では PulseAudio/ALSA を使用してマイク入力をキャプチャする。

## User Stories

### US-1: ホットキーでボイス入力を開始する

開発者として、キーボードショートカットを押すだけでマイク録音を開始し、音声でエージェントに指示を出したい。

### US-2: 録音状態を確認する

開発者として、録音中であることをステータスバーのインジケーターで一目で確認し、録音の開始・停止タイミングを把握したい。

### US-3: 変換テキストを PTY に送信する

開発者として、音声認識結果がアクティブな Shell/Agent タブの PTY に自動送信され、手動でコピー＆ペーストする必要がないこと。

## Acceptance Scenarios

### AS-1: ボイス入力の基本フロー

- gwt-tui でホットキー（Ctrl+G, v）を押す
- マイクキャプチャが開始され、ステータスバーに録音インジケーターが表示される
- ホットキーを再度押すと録音停止
- whisper-rs でテキスト変換され、アクティブ PTY に送信される

### AS-2: 録音インジケーター

- 録音中にステータスバーに赤い録音マーク（Unicode シンボル）が表示される
- 録音停止後にインジケーターが消える

### AS-3: OS 別マイクアクセス

- macOS: AVFoundation 経由でマイクアクセス（権限ダイアログ対応）
- Linux: PulseAudio または ALSA 経由でマイクアクセス

### AS-4: マイク未検出時のエラー処理

- マイクが接続されていない状態でホットキーを押す
- エラーメッセージがステータスバーに表示される

## Functional Requirements

| ID | 要件 |
|----|------|
| FR-001 | ホットキー（Ctrl+G, v）でマイクキャプチャの開始/停止を切り替える |
| FR-002 | macOS: AVFoundation を使用してマイク入力をキャプチャする |
| FR-003 | Linux: PulseAudio/ALSA を使用してマイク入力をキャプチャする |
| FR-004 | whisper-rs を使用して音声データをテキストに変換する |
| FR-005 | 変換されたテキストをアクティブな PTY に送信する |
| FR-006 | 録音中のインジケーターをステータスバーに表示する |
| FR-007 | マイク未検出・権限エラー時にエラーメッセージを表示する |

## Non-Functional Requirements

| ID | 要件 |
|----|------|
| NFR-001 | 音声変換は非同期（tokio）で実行し TUI をブロックしない |
| NFR-002 | whisper-rs モデルは初回使用時にダウンロード、以降はキャッシュ使用 |
| NFR-003 | 録音から変換完了まで 5 秒以内（30 秒以内の発話） |

## Success Criteria

| ID | 基準 |
|----|------|
| SC-001 | ホットキーでマイク録音が開始・停止される |
| SC-002 | 録音中にステータスバーにインジケーターが表示される |
| SC-003 | 音声がテキストに変換され PTY に送信される |
| SC-004 | macOS / Linux の両方でマイクアクセスが動作する |
