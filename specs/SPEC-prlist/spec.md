# 機能仕様: PR Dashboard

**仕様ID**: `SPEC-prlist`
**作成日**: 2026-02-26
**更新日**: 2026-02-26
**ステータス**: ドラフト
**カテゴリ**: GUI
**依存仕様**:

- SPEC-d6949f99（PR Status表示 - Sidebar）

**入力**: ユーザー説明: "Git menu配下にPR一覧ダッシュボードを追加し、デスクトップで完結するPR管理体験を提供"

## 背景

- 現在のGit menuにはIssues・Version History・Cleanup Worktreesがあるが、PRの一覧・管理機能がない
- 開発者はPRの状態（CI・レビュー・マージ可否）を確認するためにブラウザへ切り替える必要がある
- Sidebar PR表示はブランチコンテキストに限定されており、リポジトリ全体のPR管理はできない

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - PR一覧表示 (優先度: P0)

開発者として、リポジトリ全体のPR一覧をダッシュボードで閲覧したい。

**独立したテスト**: Git menu → Pull Requests でPR一覧タブが開き、PRが表示される

**受け入れシナリオ**:

1. **前提条件** ghが認証済み、**操作** Git menu → Pull Requestsをクリック、**期待結果** PRsタブが開きOpen PRが一覧表示される
2. **前提条件** PRsタブが開いている、**操作** Stateフィルターを切り替え、**期待結果** Open/Closed/Mergedのフィルタリングが動作する
3. **前提条件** PRが30件以上、**操作** Show Moreをクリック、**期待結果** 追加PRがロードされる

---

### ユーザーストーリー 2 - PR詳細・ステータス確認 (優先度: P0)

開発者として、各PRのCI・レビュー・マージ可否をインラインで確認したい。

**独立したテスト**: PR行にCI/レビュー/マージバッジが表示される

**受け入れシナリオ**:

1. **前提条件** PR一覧が表示されている、**操作** PR行を確認、**期待結果** CI状態・レビュー状態・マージ可否がバッジで表示される
2. **前提条件** PR一覧が表示されている、**操作** PR行をクリック、**期待結果** アコーディオン展開でPRボディがMarkdownレンダリングされる
3. **前提条件** Draft PRがある、**操作** 一覧を確認、**期待結果** Draft PRが視覚的に区別される

---

### ユーザーストーリー 3 - 自分関連PRハイライト・スマートソート (優先度: P0)

開発者として、自分が関与するPRが優先表示されることで、アクションが必要なPRを素早く見つけたい。

**独立したテスト**: 自分関連PRが視覚的に強調され、スマートソートで上位表示される

**受け入れシナリオ**:

1. **前提条件** 自分がauthorのPRがある、**操作** PR一覧を表示、**期待結果** 自分のPRが背景色で強調される
2. **前提条件** 複数のPRがある、**操作** PR一覧を表示、**期待結果** アクション要求 > 自分PR問題あり > 自分PR正常 > その他の順

---

### ユーザーストーリー 4 - PRアクション (優先度: P1)

開発者として、マージ・レビュー・Branch Update・Draft Ready等のアクションをダッシュボードから実行したい。

**独立したテスト**: マージダイアログで戦略選択・実行ができる

**受け入れシナリオ**:

1. **前提条件** マージ可能なPR、**操作** Mergeボタン → ダイアログで戦略選択・実行、**期待結果** PRがマージされる
2. **前提条件** PRが表示されている、**操作** Reviewボタン → Approve/Request Changes/Comment選択、**期待結果** レビューが投稿される
3. **前提条件** behind状態のPR、**操作** Update branchボタン、**期待結果** ブランチが更新される
4. **前提条件** Draft PR、**操作** Ready for Reviewボタン、**期待結果** PRがReadyに変更される

---

### ユーザーストーリー 5 - Worktree連携 (優先度: P1)

開発者として、PRのブランチにWorktreeでSwitchまたは新規作成したい。

**独立したテスト**: PR行からWorktree Switch/新規作成ができる

**受け入れシナリオ**:

1. **前提条件** PRのブランチにworktreeがある、**操作** Switchボタン、**期待結果** 該当worktreeにSwitchされる
2. **前提条件** PRのブランチにworktreeがない、**操作** 新規作成ボタン、**期待結果** 新規worktreeが作成される

---

### ユーザーストーリー 6 - gh未認証時ガイド (優先度: P1)

開発者として、gh CLIが未認証の場合にセットアップ手順が案内されてほしい。

**独立したテスト**: gh未認証時にセットアップガイドが表示される

**受け入れシナリオ**:

1. **前提条件** ghが未インストールまたは未認証、**操作** PRsタブを開く、**期待結果** gh CLIのセットアップガイドが表示される

## エッジケース

- PRが0件の場合: 空状態メッセージを表示
- GitHub API Rate Limit: レート制限時はキャッシュを返却、クールダウン後に再取得
- ネットワーク断: 最後のキャッシュデータを表示、エラーバナー付き
- 巨大リポジトリ（PR数千件）: 初回30件 + ページネーションで対応
- マージ戦略がリポジトリで禁止されている場合: APIエラーで判明、エラーメッセージ表示

## 要件 *(必須)*

### 機能要件

- **FR-001**: Git menu配下に"Pull Requests"メニュー項目を追加する
- **FR-002**: `gh pr list --json`で PR一覧を取得する（number, title, state, isDraft, headRefName, baseRefName, author, labels, createdAt, updatedAt, url, body）
- **FR-003**: `gh api user --jq .login`で認証ユーザーを取得しキャッシュする
- **FR-004**: State フィルター（Open/Closed/Merged）を提供する
- **FR-005**: 初回30件取得、Show Moreボタンでページネーション
- **FR-006**: 自分関連PRをハイライト表示する
- **FR-007**: スマートソート（アクション要求 > 自分PR問題あり > 自分PR正常 > その他）
- **FR-008**: PR行クリックでアコーディオン展開、PRボディをMarkdownレンダリング
- **FR-009**: CI/レビュー/マージ可否をインラインバッジ表示
- **FR-010**: マージダイアログ（merge/squash/rebase選択 + ブランチ削除 + コミットメッセージ編集）
- **FR-011**: レビューダイアログ（Approve/Request Changes/Comment + コメント本文）
- **FR-012**: Draft PRの視覚的区別 + Ready for Review変更
- **FR-013**: behind状態PRにBranch Updateインラインボタン
- **FR-014**: Worktree連携（既存→Switch / なければ→新規作成）
- **FR-015**: PR番号クリックで外部ブラウザ
- **FR-016**: タブアクティブ時30秒pollingおよび手動Refresh
- **FR-017**: gh未認証時にセットアップガイド表示

### 非機能要件

- **NFR-001**: PR一覧取得は4秒以内
- **NFR-002**: キャッシュTTL 30秒、Rate Limit時60秒バックオフ
- **NFR-003**: UIレスポンスは100ms以内

## 制約と仮定

- gh CLI がインストール・認証済みであること（未認証時はガイド表示）
- GitHub APIのRate Limitに従う
- Sidebar PRステータス表示とは独立して動作する（役割分担: Sidebar=ブランチ、Dashboard=リポジトリ全体）
- マージ戦略は常に3種類表示（禁止ならAPIエラーで判明）

## 成功基準 *(必須)*

- **SC-001**: Git menu → Pull RequestsでPR一覧タブが開き、Open PRが表示される
- **SC-002**: State フィルターでOpen/Closed/Mergedの切り替えが動作する
- **SC-003**: 自分関連PRがハイライトされスマートソートで上位表示される
- **SC-004**: PR行クリックでアコーディオン展開しMarkdown表示される
- **SC-005**: マージ・レビュー・Branch Update・Draft Ready操作が成功する
- **SC-006**: cargo clippy, svelte-check, cargo test, pnpm testがパスする
