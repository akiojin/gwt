**入力**: ユーザー説明: "サイドバーのブランチ一覧で、選択ブランチを切り替えるたびに一瞬固まるためUXを改善したい"

## 背景

- `WorktreeSummaryPanel` はブランチ切替時に複数タブ分のデータ取得を同時に開始しており、UI体感が悪化している。
- 特に PR 参照は `gh` 実行を伴うため、切替操作の連打時に重い処理が集中する。

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - ブランチ切替の即応性改善 (優先度: P0)

ユーザーとして、ブランチ一覧を選択した瞬間にUIが固まらず、選択状態が即時反映されてほしい。

**独立したテスト**: ブランチ選択直後に重いタブ向けAPI（PR/Docker/Issue）が自動発火しないことを確認する。

**受け入れシナリオ**:

1. **前提条件** Sidebar で任意ブランチを選択、**操作** 別ブランチへ連続切替、**期待結果** 選択ハイライトとヘッダーが即時更新される。
2. **前提条件** Summary タブ表示中、**操作** ブランチを切替、**期待結果** Summary 系データのみ更新し、PR/Docker/Issue は未取得のまま。

---

### ユーザーストーリー 2 - タブ表示時の遅延取得 (優先度: P1)

ユーザーとして、必要なタブを開いた時だけそのデータが取得され、不要な待ち時間が発生しないようにしたい。

**独立したテスト**: Issue/PR/Docker タブを開いたときにのみ対応コマンドが発火することを確認する。

**受け入れシナリオ**:

1. **前提条件** ブランチ選択直後、**操作** PR タブを開く、**期待結果** そのタイミングで `fetch_latest_branch_pr` が実行される。
2. **前提条件** Docker タブ未表示、**操作** 何度ブランチを切替えても Docker タブを開かない、**期待結果** `detect_docker_context` は実行されない。

## エッジケース

- PR タブ表示中にブランチを切り替えた場合、旧ブランチ応答が新ブランチ表示へ混入しない。
- 同一ブランチへ短時間で戻った場合、短期キャッシュを再利用し過剰再取得を避ける。

## 要件 *(必須)*

### 機能要件

- **FR-001**: ブランチ切替時に Summary 以外のタブデータは遅延取得する。
- **FR-002**: Issue/PR/Docker はタブ表示時に初回取得し、ブランチ単位キャッシュを持つ。
- **FR-003**: stale レスポンスを branch key/token で破棄し、表示汚染を防止する。
- **FR-004**: `fetch_latest_branch_pr` の backend 連続実行を短TTLキャッシュで抑制する。

### 非機能要件

- **NFR-001**: ブランチ切替操作で体感停止（入力待ち）を発生させない。
- **NFR-002**: 遅延取得導入後も既存の Summary ポーリング仕様（15s/60s）を維持する。

## 制約と仮定

- 既存UI構成（6タブ、ヘッダー操作）は維持し、取得タイミングのみ変更する。
- gh 未導入/未認証時の既存エラー表示仕様は変更しない。

## 成功基準 *(必須)*

- **SC-001**: ブランチ切替時に `fetch_latest_branch_pr` と `detect_docker_context` が自動実行されない。
- **SC-002**: PR/Docker/Issue タブを開いた時のみ各コマンドが実行される。
- **SC-003**: `WorktreeSummaryPanel.test.ts` の遅延取得・stale防止・既存Summary関連ケースが通過する。
