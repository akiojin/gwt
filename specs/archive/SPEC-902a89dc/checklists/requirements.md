# 要件チェックリスト: Worktreeパス修復機能

**目的**: spec.mdで定義された機能要件の実装確認
**作成日**: 2026-01-05
**機能**: [spec.md](../spec.md)

## 機能要件

- [ ] CHK001 `x`キーでWorktree修復機能が起動できる
- [ ] CHK002 チェック済みかつ`isAccessible === false`のブランチのみが修復対象になる
- [ ] CHK003 `git worktree repair`コマンドが正しく実行される
- [ ] CHK004 修復結果（成功数/失敗数）がフッターに表示される
- [ ] CHK005 修復処理中はキー入力がロックされる
- [ ] CHK006 修復処理中にフッターに進捗メッセージが表示される
- [ ] CHK007 チェックなし/修復対象なしの場合、警告メッセージが表示される
- [ ] CHK008 フィルターモード中は`x`キーが無効化される

## ユーザーシナリオ

- [ ] CHK009 🔴表示のブランチをチェック→`x`キーで修復成功
- [ ] CHK010 複数🔴ブランチの一括修復が動作する
- [ ] CHK011 修復後、🔴から🟢に表示が変わる
- [ ] CHK012 🟢ブランチをチェックしても修復対象から除外される
- [ ] CHK013 修復対象がない場合「修復対象がありません」と表示される

## エッジケース

- [ ] CHK014 チェックなしで`x`押下時、警告が表示される
- [ ] CHK015 修復失敗時、失敗情報がフッターに表示される
- [ ] CHK016 保護ブランチ（main/develop）のWorktreeも修復対象に含まれる
- [ ] CHK017 フィルターモード中、`x`キーで修復が起動しない

## 注記

- 完了した項目にチェックマーク: `[x]`
- Git 2.30.0未満の環境でのテストは環境依存のため別途確認
