# 実装計画: Worktreeパス修復機能（チェック済み対象拡張）

**仕様ID**: `SPEC-902a89dc` | **日付**: 2026-01-09 | **仕様書**: `specs/SPEC-902a89dc/spec.md`
**入力**: `specs/SPEC-902a89dc/spec.md` からの機能仕様

## 概要

- チェック済みのローカルブランチは、アクセス可能/不可能に関わらず修復対象に含める
- リモートブランチは修復対象から除外する
- 既存の進捗表示/入力ロック/結果メッセージを維持する

## 技術コンテキスト

- **言語/バージョン**: TypeScript 5.8 / Bun
- **UI**: OpenTUI (`@opentui/solid`)
- **テスト**: `bun test`（Solid UI テスト）
- **プロジェクトタイプ**: 単一プロジェクト（`src/`, `tests/`）

## 原則チェック

- 既存のブランチ一覧UIと修復フローを尊重し、最小限の変更で達成する
- CLIのユーザー向け出力は英語のみ
- 既存テストが通ることを前提にする

## 実装戦略

1. 修復対象の抽出条件を更新し、選択済みローカルブランチを対象に含める
2. 既存の進捗表示/入力ロックの動作を維持する
3. 追加テストで選択済みのアクセス可能Worktreeが対象になることを確認する

## テスト戦略

- **ユニット/UIテスト**: 修復対象の抽出が拡張されることを確認
- **回帰**: 既存の修復フローが破壊されないことを確認
