# 機能仕様: Version History の最新タグ表示を安定化

**仕様ID**: `SPEC-1242`  
**作成日**: 2026-02-26  
**更新日**: 2026-02-26  
**ステータス**: ドラフト  
**カテゴリ**: GUI / Git  
**依存仕様**: なし  

**入力**: ユーザー説明: "Issue #1242を解決して下さい。"

## 背景

- Git > Version History で表示されるタグ一覧が、環境によっては最新順にならず古いタグから表示されることがある。
- 実際に `v7.12.x` より `v7.9.0` などが先に表示され、最新10件が見えない状態が発生している。

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - 最新バージョンを先頭で確認したい (優先度: P0)

ユーザーとして、Version History を開いたときに、常に最新のバージョンタグ10件を先頭から確認したい。

**独立したテスト**: タグ文字列の入力順に依存せず、セマンティックバージョン降順で並ぶことをユニットテストで検証する。

**受け入れシナリオ**:

1. **前提条件** タグに `v7.12.6` と `v7.9.0` が存在する、**操作** Version History を表示する、**期待結果** `v7.12.6` が `v7.9.0` より先に表示される。
2. **前提条件** タグが10件を超える、**操作** Version History を表示する、**期待結果** 最新順の先頭10件のみが対象になる。

---

### ユーザーストーリー 2 - 不正/非標準タグが混在しても壊れない (優先度: P1)

ユーザーとして、バージョン形式でないタグが混在していても Version History が安定して表示されてほしい。

**独立したテスト**: 非semverタグ混在時のソートが決定的であり、処理が失敗しないことをユニットテストで検証する。

## エッジケース

- semverとして解釈できないタグが混在しても、処理全体は失敗しない。
- prerelease や build metadata を含むタグでも順序が破綻しない。

## 要件 *(必須)*

### 機能要件

- **FR-001**: Version History のタグ一覧は、Git の `--sort` 動作に依存せず semver 降順で整列されなければならない。
- **FR-002**: semver として解釈できないタグが混在する場合でも、処理は成功し安定した順序を返さなければならない。
- **FR-003**: GUI が利用する先頭10件は、整列後のタグ列から切り出されなければならない。

### 非機能要件

- **NFR-001**: 既存の `list_project_versions` API 互換性（レスポンス形式）を維持する。
- **NFR-002**: タグ整列処理は O(n log n) 以内で完了する。

## 制約と仮定

- バージョンタグは `v` プレフィックス（例: `v7.12.6`）を前提とする。
- 実際の表示件数制御（10件）は既存 GUI 実装のまま維持する。

## 成功基準 *(必須)*

- **SC-001**: `version_history` のユニットテストで semver 降順ソートが検証される。
- **SC-002**: `list_project_versions` が最新タグを先頭で返すことをテストで確認できる。
- **SC-003**: 変更範囲のテスト（`cargo test` の対象テスト）が通過する。
