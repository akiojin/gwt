### 技術コンテキスト

- 現行 Rust 実装のインデックス・検索パイプライン（ChromaDB ベース）を C# に移植する
- SQLite-vec を唯一のベクトル DB 候補として検証・導入する
- AI API（#1550）経由でベクトル埋め込みを生成する

### 実装アプローチ

- SQLite-vec 一本で検証を進め、動作不可時にのみ代替ソリューションを検討する
- 全利用者（Lead、Agent、ユーザーUI）が同一インデックスを共有する設計

### フェーズ分割

1. 既存 Rust 実装のインデックス・検索パイプラインを分析する
2. C# の `IIndexService` インターフェースを設計する
3. SQLite-vec の C# バインディングを選定・導入し、Unity環境での動作検証を行う
4. ファイルスキャン・テキスト抽出ロジックを実装する
5. ベクトル埋め込み生成（AI API 経由）を実装する
6. ベクトル類似度検索を実装する
7. GitHub Issues インデックスを実装する
8. バックグラウンドインデックスの非同期実行を実装する
9. 差分インデックス（変更検出・部分更新）を実装する
10. VContainer での DI 登録を行う
11. テストを作成する
