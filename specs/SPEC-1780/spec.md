# ファイル貼り付け（クリップボードから PTY へ）

## Background

gwt-tui に専用ショートカットでクリップボードのファイル情報を検出し、ファイルパスを PTY に送信する機能を追加する。ファイルの内容を base64 エンコードで送信するのではなく、ファイルパスを文字列として PTY に挿入する方式を採用する。

## User Stories

### US-1: クリップボードのファイルを PTY に送信する

開発者として、Finder/ファイルマネージャでコピーしたファイルを専用ショートカットで PTY に送信し、エージェントにファイルパスを渡したい。

### US-2: 複数ファイルを一括送信する

開発者として、複数ファイルをクリップボードにコピーした状態で、全ファイルパスを一括で PTY に送信したい。

## Acceptance Scenarios

### AS-1: 単一ファイルペースト

- Finder でファイルをコピー（Cmd+C）
- gwt-tui で Ctrl+G, p を押す
- ファイルの絶対パスが PTY に送信される

### AS-2: 複数ファイルペースト

- Finder で複数ファイルを選択してコピー
- gwt-tui で Ctrl+G, p を押す
- 各ファイルの絶対パスが改行区切りで PTY に送信される

### AS-3: テキストクリップボード時のフォールバック

- テキストのみがクリップボードにある状態で Ctrl+G, p を押す
- ファイルが検出されない旨のメッセージがステータスバーに表示される

### AS-4: OS 別クリップボードアクセス

- macOS: NSPasteboard API でファイル URL を取得
- Linux: xclip または wl-paste でファイルパスを取得

## Functional Requirements

| ID | 要件 |
|----|------|
| FR-001 | 専用ショートカット（Ctrl+G, p）でクリップボードのファイル情報を検出する |
| FR-002 | macOS: NSPasteboard API（`NSFilenamesPboardType`）でファイル URL を取得する |
| FR-003 | Linux: xclip（X11）または wl-paste（Wayland）でファイルパスを取得する |
| FR-004 | 検出されたファイルパスを絶対パスとして PTY に送信する |
| FR-005 | 複数ファイルの場合は改行区切りで全パスを送信する |
| FR-006 | ファイルが検出されない場合はステータスバーにメッセージを表示する |

## Non-Functional Requirements

| ID | 要件 |
|----|------|
| NFR-001 | クリップボードアクセスは非同期で TUI をブロックしない |
| NFR-002 | パスにスペースや特殊文字が含まれる場合はクォートで囲む |

## Success Criteria

| ID | 基準 |
|----|------|
| SC-001 | Ctrl+G, p でクリップボードのファイルパスが PTY に送信される |
| SC-002 | 複数ファイルが改行区切りで送信される |
| SC-003 | macOS / Linux の両方で動作する |
| SC-004 | テキストのみクリップボード時にフォールバックメッセージが表示される |
