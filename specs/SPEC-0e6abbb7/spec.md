# 機能仕様: 全画面テキストコピー (Cmd+Shift+C)

**仕様ID**: `SPEC-0e6abbb7`
**作成日**: 2026-02-18
**更新日**: 2026-02-18
**ステータス**: ドラフト
**カテゴリ**: GUI
**依存仕様**: なし

**入力**: ユーザー説明: "Cmd+Shift+Cで、全画面のテキストコピーに対応したい。目的は、問題が起きた場合にAIに貼り付けて伝えること。"

## 背景

- gwt の画面に表示されている情報（ブランチ名、ターミナル出力、Git情報等）を AI に伝えて問題を報告する際、個別にテキストを選択・コピーする手間がかかる
- 画面全体の可視情報を一度に構造化テキストとしてコピーできれば、AI が問題を素早く正確に理解できる
- 既存の Cmd+C は選択テキストのコピーに使用されており、全画面コピーは別ショートカットが必要

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - ターミナルエラーの報告 (優先度: P0)

ユーザーとして、ターミナルでエラーが発生した際に Cmd+Shift+C を押して画面全体の情報を構造化テキストとしてコピーし、AI チャットに貼り付けて問題を報告したい。

**独立したテスト**: ターミナルタブがアクティブな状態で Cmd+Shift+C を押し、クリップボードに構造化テキストがコピーされることを確認

**受け入れシナリオ**:

1. **前提条件** ターミナルタブがアクティブでコマンド出力が表示されている、**操作** Cmd+Shift+C を押す、**期待結果** クリップボードにセクション区切り付きの構造化テキストがコピーされる（ヘッダーにブランチ名・アクティブタブ・ウィンドウサイズ、サイドバー・ターミナル・ステータスバーの各セクション）
2. **前提条件** 上記の操作を実行、**操作** テキストエディタに貼り付ける、**期待結果** ターミナル出力が ANSI コード除去済みのプレーンテキストで含まれている
3. **前提条件** 上記の操作を実行、**操作** 画面を確認する、**期待結果** アクセントカラー（青系）の半透明オーバーレイが約0.2秒でフェードイン・アウトし、トーストで "Copied to clipboard" と表示される

---

### ユーザーストーリー 2 - 非ターミナルパネルの報告 (優先度: P0)

ユーザーとして、Git Changes や Worktree Summary などの非ターミナルパネルを表示中に問題が起きた際も、同じ操作で画面情報をコピーしたい。

**独立したテスト**: Git Changes タブがアクティブな状態で Cmd+Shift+C を押し、パネルの可視テキストがコピーされることを確認

**受け入れシナリオ**:

1. **前提条件** Git Changes タブがアクティブ、**操作** Cmd+Shift+C を押す、**期待結果** メインエリアセクションにアクティブパネルの可視テキストが含まれる
2. **前提条件** Settings パネルが表示されている、**操作** Cmd+Shift+C を押す、**期待結果** Settings パネルの可視テキストがコピーされる

---

### ユーザーストーリー 3 - モーダルダイアログのエラー報告 (優先度: P1)

ユーザーとして、モーダルダイアログにエラーが表示されている状態でも画面情報をコピーし、ダイアログの内容と背景の画面情報の両方を AI に伝えたい。

**独立したテスト**: モーダルが開いた状態で Cmd+Shift+C を押し、モーダル内容と背景画面の両方がコピーされることを確認

**受け入れシナリオ**:

1. **前提条件** Agent Launch Form モーダルが表示されている、**操作** Cmd+Shift+C を押す、**期待結果** "--- Modal ---" セクションにモーダル内テキストが含まれ、背景の画面情報も通常通り含まれる
2. **前提条件** 削除確認ダイアログが表示されている、**操作** Cmd+Shift+C を押す、**期待結果** ダイアログ内のエラーメッセージや警告文がモーダルセクションに含まれる

## エッジケース

- クリップボード API が利用不可の場合（権限拒否等）: トーストでエラーを表示
- サイドバーが折り畳まれている場合: サイドバーセクションを省略
- 複数のモーダルが重なっている場合: 最前面のモーダルのみをモーダルセクションに含める
- ウィンドウが極端に小さい場合: 可視テキストが少なくても正常にコピー
- ターミナル出力が空の場合: "--- Terminal ---" セクション内を "(empty)" と表示

## 要件 *(必須)*

### 機能要件

- **FR-001**: Cmd+Shift+C（macOS）/ Ctrl+Shift+C（Windows/Linux）のキーボードショートカットで全画面テキストコピーを実行する
- **FR-002**: コピー内容は以下の構造化テキスト形式とする:

```text
=== GWT Screen Capture ===
Branch: {current_branch}
Active Tab: {tab_name}
Window: {width}x{height}

--- Sidebar ---
{sidebar_visible_text}

--- Main: {active_tab_name} ---
{main_area_visible_text}

--- Status Bar ---
{status_bar_text}
```

- **FR-003**: ターミナル出力は可視行のみを対象とし、ANSI エスケープシーケンスを除去したプレーンテキストとする
- **FR-004**: モーダルダイアログが開いている場合、"--- Modal ---" セクションをメインセクションの前に追加し、モーダル内テキストと背景画面の両方をコピーする
- **FR-005**: コピー完了時にアクセントカラー（--accent: #89b4fa）の半透明オーバーレイを約0.2秒でフェードイン・アウトし、トーストで "Copied to clipboard" と表示する
- **FR-006**: サイドバーの表示テキストはそのまま取得する（追加メタデータなし）
- **FR-007**: Tauri ネイティブメニューの Edit メニューに "Copy Screen Text" (Cmd+Shift+C) を追加する

### 非機能要件

- **NFR-001**: コピー処理は100ms以内に完了すること（ユーザー体感でラグがないこと）
- **NFR-002**: 機密情報のマスキングは行わない（ユーザーの責任で判断）

## 制約と仮定

- xterm.js の buffer API を使用してターミナルの可視行テキストを取得する
- DOM の innerText / textContent を使用してUI要素のテキストを取得する
- navigator.clipboard.writeText() API を使用する
- Tauri ネイティブメニューでショートカットを登録し、フロントエンドにイベントを送信する

## 成功基準 *(必須)*

- **SC-001**: Cmd+Shift+C で画面全体の構造化テキストがクリップボードにコピーされる
- **SC-002**: コピーされたテキストに全セクション（ヘッダー、サイドバー、メイン、ステータスバー）が含まれる
- **SC-003**: ターミナル出力に ANSI コードが含まれない
- **SC-004**: モーダル表示時にモーダル内容も含まれる
- **SC-005**: 視覚フィードバック（フラッシュ + トースト）が表示される
