# 機能仕様: GitHub リモート起点の Worktree 作成

**仕様ID**: `SPEC-a4fb2db2`
**作成日**: 2026-02-21
**更新日**: 2026-02-21
**ステータス**: 実装済み
**カテゴリ**: GUI
**依存仕様**:

- SPEC-c6ba640a（GitHub Issue連携によるブランチ作成）

**入力**: ユーザー説明: "新規ブランチ作成時に GitHub 上にリモートブランチを先に作成し、それを fetch して Worktree にする"

## 背景

- 現在、新規ブランチの Worktree 作成は `git worktree add -b` でローカルに直接作成している
- ユーザーは GitHub 上でブランチを先に作成し、それを fetch して Worktree にするフロー（リモート起点）を求めている
- 既存の `gh` CLI 統合（`gh_cli.rs`）のインフラを再利用する

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - リモート起点でブランチ作成 (優先度: P0)

ユーザーとして、新規ブランチ作成時に GitHub 上にリモートブランチを先に作成し、それを fetch して Worktree にしたい。

**独立したテスト**: `gh` CLI が利用可能な環境で新規ブランチ名を指定して Worktree を作成し、GitHub 上にブランチが存在することを確認する。

**受け入れシナリオ**:

1. **前提条件** `gh` CLI がインストール済みかつ認証済み、**操作** 新規ブランチ名を指定して Worktree を作成、**期待結果** GitHub 上にブランチが作成され、ローカルに fetch されて Worktree が作成される
2. **前提条件** `gh` CLI が認証済み、ベースブランチに `develop` を指定、**操作** 新規ブランチ作成、**期待結果** `develop` の SHA をベースにリモートブランチが作成される

---

### ユーザーストーリー 2 - gh 未利用時のフォールバック (優先度: P1)

ユーザーとして、`gh` CLI が未インストールまたは未認証の場合でも、従来のローカルブランチ作成で Worktree を作成したい。

**独立したテスト**: `gh` CLI が利用不可の環境で新規ブランチ名を指定して Worktree を作成し、ローカルにブランチが作成されることを確認する。

**受け入れシナリオ**:

1. **前提条件** `gh` CLI 未インストール、**操作** 新規ブランチ名を指定して Worktree を作成、**期待結果** 従来のローカルブランチ作成で Worktree が作成される
2. **前提条件** `gh` CLI インストール済みだが未認証、**操作** 新規ブランチ名を指定、**期待結果** warn ログ出力後、ローカルフォールバックで Worktree 作成
3. **前提条件** ネットワークタイムアウト発生、**操作** 新規ブランチ名を指定、**期待結果** warn ログ出力後、ローカルフォールバックで Worktree 作成

---

### ユーザーストーリー 3 - Issue リンクとの互換性 (優先度: P2)

ユーザーとして、Issue リンク付きブランチ作成（SPEC-c6ba640a）とリモート起点フローが共存し、`gh issue develop` が正常に動作することを確認したい。

**独立したテスト**: リモート起点で作成したブランチに対して `gh issue develop` で Issue リンクが正常に設定されることを確認する。

**受け入れシナリオ**:

1. **前提条件** リモート起点でブランチ作成済み、**操作** `gh issue develop` で Issue リンク、**期待結果** 既存リモートブランチに Issue がリンクされる

## エッジケース

- **422 ブランチ既存**: GitHub API が 422 を返した場合、エラーを返す（フォールバックしない。SPEC-e1004use の E1004 UX と連携）
- **ネットワークエラー**: タイムアウト / 403 / 404 の場合、ローカルフォールバック
- **ベアリポジトリ**: ベースブランチの SHA 解決ロジックはリモート参照を使用
- **fetch 失敗（リモート作成済み）**: リモートブランチ作成後に fetch が失敗した場合、エラーを返す（リモートブランチは残る旨をエラーメッセージで案内）

## 要件 *(必須)*

### 機能要件

- **FR-001**: `create_remote_branch()` — GitHub API 経由でリモートブランチを作成する
- **FR-002**: `resolve_remote_branch_sha()` — GitHub API 経由でブランチの SHA を取得する
- **FR-003**: `create_new_worktree_path()` で `gh` 利用可能時にリモート起点フローを使用する
- **FR-004**: リモート起点フロー失敗時（422 以外）にローカルフォールバックする
- **FR-005**: 422（ブランチ既存）エラーはフォールバックせずエラーを返す
- **FR-006**: `git fetch origin branch:branch` でローカルに取得する

### 非機能要件

- **NFR-001**: GitHub API 呼び出しのタイムアウトは 10 秒
- **NFR-002**: フォールバック時に `tracing::warn` でログを出力する

## 制約と仮定

- `gh` CLI がインストール済みかつ認証済みであることがリモート起点フローの前提
- GitHub API の `git/refs` エンドポイントを使用する
- 既存の `resolve_owner_repo()` / `wait_with_timeout()` を再利用する

## 成功基準 *(必須)*

- **SC-001**: `cargo test` で新規テスト + 既存テスト全パス

- **SC-003**: `gh` 利用可能環境で新規ブランチ作成時に GitHub 上にブランチが存在する
- **SC-004**: `gh` 未利用環境で従来のローカル作成にフォールバックする
