### 背景
Issue検索フィルターでIssue番号を入力しても一致せず、既存Issueの絞り込み効率が下がっている。

### ユーザーシナリオ
- P0: ユーザーが Issue 一覧/From Issue で `123` を入力すると `#123` を含む Issue がヒットする。
- P0: ユーザーが `bug 12` を入力すると、タイトルに `bug` を含み、番号に `12` を含む Issue のみヒットする。
- P1: `#12` 入力でも `12` と同じ結果になる。

### 機能要件
- FR-001: Issue検索はタイトル部分一致（大文字小文字無視）を維持する。
- FR-002: Issue検索は番号トークン（数字または `#` + 数字）を番号部分一致として扱う。
- FR-003: 複数トークン検索は AND 条件で評価する。
- FR-004: IssueListPanel と AgentLaunchForm の検索挙動を統一する。
- FR-005: Rust の `filter_issues_by_title` も同等仕様をサポートする。

### 非機能要件
- NFR-001: 既存のラベルフィルター・無限スクロール挙動を回帰させない。
- NFR-002: 既存公開インターフェースの破壊的変更を行わない。

### 成功基準
- SC-001: `12` で `#123` がヒットするテストが GUI/Rust 両方で通る。
- SC-002: `bug 12` の AND 条件テストが GUI/Rust 両方で通る。
- SC-003: 既存のタイトル検索テスト・ラベルフィルターテストが通る。
