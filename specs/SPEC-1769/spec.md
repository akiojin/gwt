# spec.md — Agent Canvas 画像ビューアタイル

## Background

Agent Canvas 上に画像を表示するためのタイルタイプを追加する。
エージェントが生成したスクリーンショット、アーキテクチャ図、UI モックアップなどをキャンバス上で直接確認できるようにすることで、コンテキスト切り替えを減らし、エージェントとの協調作業を効率化する。

## User Stories

### US-1: 画像ファイルをキャンバスに配置する

ユーザーとして、ローカルの画像ファイルを Agent Canvas 上にドラッグ＆ドロップまたはファイル選択で配置し、他のタイル（テキスト・ターミナル等）と並べて参照したい。

### US-2: エージェント出力の画像を表示する

ユーザーとして、エージェントが生成・取得した画像（スクリーンショット、図表など）を自動的にキャンバス上の画像タイルとして表示し、テキスト出力と合わせて確認したい。

### US-3: ズーム・パンで画像を詳細確認する

ユーザーとして、画像タイル内で画像をズーム・パン操作し、細部を確認したい。タイルのリサイズとは独立して画像の表示倍率を制御できることを期待する。

### US-4: クリップボードから画像を貼り付ける

ユーザーとして、クリップボードにコピーした画像をキャンバスに貼り付けて、即座に画像タイルとして表示したい。

### US-5: URL 参照の画像を表示する

ユーザーとして、外部 URL の画像をキャンバス上の画像タイルとして参照・表示したい。

## Clarifications (Resolved)

全項目がユーザーとの clarify セッションで確定済み。

- **対応画像フォーマット**: PNG / JPG / SVG / WebP / GIF / BMP の全フォーマットをサポートする
- **画像ソース**: ローカルファイル、クリップボード貼り付け、URL 参照の 3 ソースをサポートする
- **画像タイル内ズーム**: タイル内で画像の拡大縮小・パン操作が可能（タイルリサイズとは独立）
- **画像の永続化**: デフォルトはファイルパス参照。クリップボード貼り付け時は `~/.gwt/images/` にコピー保存する（両方式サポート）
- **worktree との relation edge**: 手動紐付け可能（メモタイルと同じ方式）

## Functional Requirements

- FR-1: 画像タイルの作成・配置 — ローカルファイルのドラッグ＆ドロップ、ファイル選択ダイアログ、クリップボード貼り付け（Ctrl+V / Cmd+V）、URL 指定の 4 つの入力方式で画像タイルを作成できること（US-1, US-4, US-5 に対応）
- FR-2: 対応フォーマット — PNG, JPG (JPEG), SVG, WebP, GIF, BMP の読み込み・表示をサポートすること
- FR-3: エージェント出力からの画像タイル自動生成 — エージェントが生成・取得した画像を自動的にキャンバス上の画像タイルとして配置すること（US-2 に対応）
- FR-4: 画像のズーム・パン操作 — タイル内で画像の拡大縮小（ホイール / ピンチ）とパン（ドラッグ）が可能であること。タイルのリサイズとは独立して動作すること（US-3 に対応）
- FR-5: ズームリセット — ダブルクリックまたはリセットボタンでフィット表示に戻せること
- FR-6: 画像の永続化 — ローカルファイルおよび URL はパス／URL 参照で保存する。クリップボード貼り付け画像は `~/.gwt/images/` にコピー保存し、そのパスを参照する
- FR-7: worktree relation edge — メモタイルと同じ方式で画像タイルを worktree に手動紐付けできること

## Non-Functional Requirements

- NFR-1: 大きな画像ファイル（50 MB 以上）でもタイル表示がブロッキングしないこと（非同期読み込み・サムネイル生成）
- NFR-2: 画像タイルのズーム・パン操作が他のタイルのパフォーマンスに影響しないこと（60 fps を維持）
- NFR-3: SVG はベクタ表示し、ズーム時にラスタライズによる劣化が発生しないこと
- NFR-4: GIF アニメーションはタイル内で再生可能であること
- NFR-5: `~/.gwt/images/` に保存するクリップボード画像はタイムスタンプ付きファイル名で重複を避けること

## Success Criteria

- SC-1: PNG / JPG / SVG / WebP / GIF / BMP の各フォーマットの画像ファイルをキャンバスに配置し、正しく表示できること
- SC-2: ドラッグ＆ドロップ、ファイル選択、クリップボード貼り付け、URL 指定の全入力方式で画像タイルを作成できること
- SC-3: エージェント出力の画像がキャンバス上に自動表示されること
- SC-4: タイル内でズーム・パン操作により画像の詳細を確認でき、ダブルクリックでフィット表示にリセットできること
- SC-5: クリップボード貼り付け画像が `~/.gwt/images/` に保存され、キャンバス再読み込み後も表示されること
- SC-6: 画像タイルを worktree に手動紐付けでき、relation edge が表示されること
