# 実装計画: ウィンドウタイトルにアプリバージョンを表示（GUI）

## 目的

- 実行中の gwt バージョンをタイトルバーから即座に判別できるようにする
- 非 Tauri 環境（web preview 等）でも例外で落ちないフォールバックを維持する

## 実装方針

### Phase 1: タイトル文字列生成の分離

- タイトルのフォーマットをフロントエンドのユーティリティへ切り出す
- 文字列生成をユニットテストで固定し、回帰を防止する

### Phase 2: バージョン取得（best-effort）

- Tauri API からアプリバージョンを取得する
- 取得失敗時は `null` として扱い、従来のタイトル形式へフォールバックする

### Phase 3: ウィンドウタイトル更新

- `document.title` を更新しつつ、Tauri 実行時は `setTitle()` でネイティブのタイトルバーも更新する
- バージョン取得は UI 表示をブロックしない（取得完了後にタイトルを更新）

## テスト

- unit test（Vitest）でタイトルフォーマットを検証する
- `pnpm -C gwt-gui test` が成功すること
- `pnpm -C gwt-gui check`（svelte-check）でエラー・警告がないこと
