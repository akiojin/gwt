# 機能仕様: ウィンドウタイトルにアプリバージョンを表示（GUI）

**仕様ID**: `SPEC-ab9b7d08`  
**作成日**: 2026-02-10  
**ステータス**: 実装完了  
**カテゴリ**: GUI  
**入力**: ユーザー説明: "gwtアプリバージョンをタイトルに表示"

## 背景

現在の gwt GUI は、ウィンドウタイトルに `gwt` または `gwt - <projectPath>` を表示している。
不具合報告・スクリーンショット共有・複数バージョン併用の状況では、**実行中バージョンが即座に判別できない**ため、調査コストが上がる。

よって、Tauri 実行環境で取得できるアプリバージョンを、ウィンドウタイトルに常時含める。

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - 起動時にバージョンがタイトルへ表示される (優先度: P1)

開発者として、起動直後のタイトルバーから gwt のバージョンを即座に把握したい。

**受け入れシナリオ**:

1. **前提条件** gwt GUI を起動し、まだプロジェクト未選択、**操作** タイトルバーを確認する、**期待結果** タイトルが `gwt v<version>` 形式になる
2. **前提条件** Open Project でプロジェクトを開く、**操作** タイトルバーを確認する、**期待結果** タイトルが `gwt v<version> - <projectPath>` 形式になる

### ユーザーストーリー 2 - 非Tauri環境では従来形式にフォールバックする (優先度: P2)

開発者として、Web preview 等の非 Tauri 環境でもタイトル更新処理が例外で落ちないことを保証したい。

**受け入れシナリオ**:

1. **前提条件** Tauri API が利用できない環境で UI を表示、**操作** タイトルを確認する、**期待結果** 例外が発生せず、タイトルが `gwt` または `gwt - <projectPath>` の従来形式になる

### エッジケース

- バージョン取得が失敗した場合でも UI 初期表示はブロックされず、タイトルは従来形式へフォールバックすること
- タイトルはユーザー向け表示として英語のみで構成されること（例: `gwt v6.30.3 - /path/to/repo`）

## 要件

### 機能要件

- **FR-460**: システムは、Tauri 実行環境で取得できるアプリバージョンをウィンドウタイトルに含めなければ**ならない**
- **FR-461**: プロジェクト未選択時のタイトルは `gwt v<version>`、プロジェクト選択時は `gwt v<version> - <projectPath>` の形式で表示しなければ**ならない**
- **FR-462**: バージョンが取得できない場合、システムは例外を投げては**ならず**、従来形式（`gwt` / `gwt - <projectPath>`）へフォールバックしなければ**ならない**

### 非機能要件

- **NFR-460**: バージョン取得は best-effort とし、UI の初期表示をブロックしては**ならない**
