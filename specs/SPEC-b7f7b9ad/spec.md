# 機能仕様: gwt メニューショートカットと終了保護

**仕様ID**: `SPEC-b7f7b9ad`
**作成日**: 2026-02-13
**ステータス**: ドラフト
**カテゴリ**: GUI
**依存仕様**: `SPEC-4470704f`（Native メニュー）, `SPEC-dfb1611a`（トレイ常駐と終了制御）
**入力**: ユーザー説明: "Cmd+Q が終了してしまう、Cmd+C/V が自然に動かない"

## 機能要件

- **FR-101**: `Cmd+Q`（macOS）で `trap` される終了要求時、アプリ本体は即時終了せず、実行中エージェントがある場合は確認を出して `Close` のみ可能にする。
- **FR-102**: エージェント未実行時、`Cmd+Q` はアプリを終了せず、対象ウィンドウを非表示（hide）し継続稼働する。
- **FR-103**: ネイティブ Edit メニュー `Copy/Paste` は、キーボードショートカットからと同様に、現在アクティブなターミナル pane に対して直接コピー/ペーストを実行する。
- **FR-104**: `Cmd/Ctrl + C` は、端末選択文字列がある場合は `copy` を、ない場合は既存の中断送信（`0x03`）と整合した従来動作を保つ。`Cmd + C` のみ選択なし時は `0x03` を送信せず、コピー意図を優先する。
- **FR-105**: 端末ペインでのペーストは、macOS では `Cmd+V`、Windows/Linux では `Ctrl+Shift+V`（および既存条件）時に、ペースト内容を端末入力として送信する。Windows/Linux の `Ctrl+V` は端末アプリ側へ素通しし、互換動作（例: Codex の画像貼り付け）を維持する。
- **FR-106**: 非アクティブなターミナルペインはキーボード/クリップボードのペーストイベントを処理しない
- **FR-107**: 非アクティブなターミナルペインはDOMフォーカスを保持しない（タブ切り替え時に blur する）
- **FR-108**: 編集可能要素（input/textarea/contenteditable）にフォーカスがある場合、メニュー経由のペーストはターミナルではなくその要素を優先する
- **FR-109**: ターミナルのフォーカス管理はアクティブターミナルのトラックパッド/マウス操作に影響を与えない

## ユーザーストーリー

- **US-101**: 開発者として、ターミナルで重要な操作中でも `Cmd+Q` で誤終了しない。
- **US-102**: 開発者として、ネイティブメニューの Copy/Paste で端末操作をマウスなしで実行できる。
- **US-103**: 開発者として、複数ターミナルタブがある時、ペーストが常にアクティブなタブに反映される
- **US-104**: 開発者として、サイドバーの入力フィールドにペーストできる

## 受け入れシナリオ

1. **前提条件** 実行中エージェントあり、**操作** macOS の `Cmd+Q`、**期待結果** 確認ダイアログが表示され、Cancel で継続、Close でウィンドウ非表示。
2. **前提条件** エージェント未実行、**操作** `Cmd+Q`、**期待結果** アプリは終了せずウィンドウが非表示。
3. **前提条件** ターミナルで文字を選択、**操作** メニュー `Edit > Copy` または `Cmd/Ctrl + C`、**期待結果** 選択文字がクリップボードに入る。
4. **前提条件** クリップボードに文字、**操作** メニュー `Edit > Paste` または（macOS:`Cmd+V` / Windows/Linux:`Ctrl+Shift+V`）、**期待結果** 文字が端末へ貼り付け。
5. **前提条件** エージェントタブAとターミナルタブBが存在しタブBがアクティブ、**操作** ペーストショートカット、**期待結果** タブBにペーストされタブAには影響なし
6. **前提条件** サイドバーの入力フィールドにフォーカス、**操作** ペーストショートカット、**期待結果** 入力フィールドにペーストされる
7. **前提条件** タブ切り替え直後、**操作** トラックパッドスクロール、**期待結果** アクティブターミナルで正常にスクロールする

## 制約

- UI文言は英語。
- 既存のカット/ペースト関連のショートカット競合（`Cut`/`Select All`）を破壊しない。
- フロント/バックのイベント名は既存の `menu-action` と `terminal-output` を流用。
