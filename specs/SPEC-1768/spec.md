# spec.md — Agent Canvas タイルシステム共通仕様

## Background

Agent Canvas に複数のタイルタイプ（assistant, worktree, agent, terminal, editor, memo, image 等）が追加されていくが、タイルの共通インターフェースが未定義。新タイルタイプ追加時に毎回同じ設計判断（デフォルトサイズ、永続化、viewport 外挙動、relation edge）を繰り返している。タイルシステムの共通仕様を定義し、拡張性の基盤を作る。

## User Stories

1. **開発者として**、新しいタイルタイプを追加する際に共通仕様に従うことで、一貫した UX を保証したい
2. **ユーザーとして**、全てのタイルが同じ操作体系（ドラッグ、リサイズ、削除、relation edge）で動作してほしい
3. **開発者として**、タイルタイプごとの viewport 外挙動を宣言的に定義し、パフォーマンスとリソース管理を適切に行いたい
4. **ユーザーとして**、コンテキストメニューから共通操作（削除、edge 接続、サイズリセット）とタイプ固有操作の両方にアクセスしたい

## Acceptance Scenarios

### AS-1: タイルサイズ制約の適用

- タイルタイプごとに定義された min/max/default サイズが適用される
- リサイズ操作時に min/max の範囲内に制約される
- 新規タイル作成時にタイプ固有の default サイズで配置される

### AS-2: viewport 外挙動

- ターミナルタイルが viewport 外に移動した場合、xterm.js インスタンスがアンマウントされる
- エディタタイルが viewport 外に移動した場合、CodeMirror インスタンスがアンマウントされる
- メモタイルが viewport 外に移動した場合、そのまま表示が維持される（軽量 DOM のため）
- viewport 内に戻った際、アンマウントされたタイルは状態を復元して再マウントされる

### AS-3: relation edge の接続

- worktree → session の edge は自動的に生成される
- それ以外のタイル間（例: memo → worktree）は手動で edge を紐付けできる
- edge の追加・削除はコンテキストメニューまたはドラッグ操作で行える

### AS-4: コンテキストメニュー

- 全タイル共通メニュー項目: 削除、edge 接続、サイズリセット
- タイルタイプごとの固有メニュー項目が共通メニューに追加される（例: ターミナル → セッションリセット、エディタ → ファイル保存）
- 共通メニューが先に表示され、タイプ固有項目がセパレータの後に続く

## Functional Requirements

### FR-1: タイル共通プロパティ

全タイルは以下の必須プロパティを持つ:

- **id**: タイルの一意識別子（UUID）
- **type**: タイルタイプ識別子（`"assistant"`, `"worktree"`, `"terminal"`, `"editor"`, `"memo"`, `"image"` 等）
- **geometry**: 位置とサイズ（x, y, width, height）
- **title**: タイルのヘッダー表示名
- **zIndex**: タイルの重なり順序
- **visible**: タイルの表示/非表示状態
- **locked**: タイルの移動・リサイズのロック状態

タイルタイプ固有のプロパティは各タイプの拡張として定義する（例: worktree タイルは worktreeId を追加で持つ）。

### FR-2: タイルサイズ制約

タイルタイプごとに以下のサイズ制約を個別定義する:

- **minWidth / minHeight**: リサイズ可能な最小サイズ
- **maxWidth / maxHeight**: リサイズ可能な最大サイズ
- **defaultWidth / defaultHeight**: 新規作成時の初期サイズ

各タイルタイプの制約はタイプ登録時に宣言的に定義する。

### FR-3: viewport 外挙動

タイルタイプごとに viewport 外の挙動を宣言的に定義する:

- **unmount**: viewport 外でコンポーネントをアンマウントし、リソースを解放する（ターミナル、エディタ等の重量コンポーネント向け）
- **keep**: viewport 外でもそのまま DOM を維持する（メモ等の軽量コンポーネント向け）

アンマウント対象のタイルは、viewport 内に戻った際に内部状態を復元して再マウントする。状態の保存・復元はタイルタイプの責務として定義する。

### FR-4: relation edge モデル

- 自動 edge: 特定のタイルペア間で自動的に生成される（worktree → session）
- 手動 edge: ユーザー操作により任意のタイル間で紐付け可能
- edge には方向性がある（source → target）
- edge の視覚表現はタイル間を結ぶ線（直線またはベジェ曲線）

### FR-5: コンテキストメニュー

- 共通メニュー項目（全タイルで利用可能）:
  - 削除
  - edge 接続（ドラッグ開始）
  - サイズリセット（default サイズに戻す）
- タイルタイプ固有メニュー項目:
  - タイルタイプ登録時に追加メニュー項目を宣言的に定義
  - 共通メニューの後にセパレータを挟んで表示

### FR-6: タイルタイプ登録

新しいタイルタイプの追加は、以下の情報を宣言的に登録する方式とする:

- タイプ識別子
- サイズ制約（FR-2）
- viewport 外挙動（FR-3）
- 自動 edge ルール（FR-4、該当する場合）
- コンテキストメニュー拡張項目（FR-5、該当する場合）
- タイプ固有プロパティの定義

## Non-Functional Requirements

### NFR-1: パフォーマンス

- viewport 外のアンマウント対象タイルは、viewport 判定から 100ms 以内にアンマウントを開始する
- viewport 内への復帰時、再マウントと状態復元は 200ms 以内に完了する
- 同時表示タイル数 50 以上でも Canvas 操作（パン・ズーム）が 60fps を維持する

### NFR-2: 拡張性

- 新しいタイルタイプの追加は、共通仕様の変更なしにタイプ登録のみで完結する
- タイプ固有の振る舞い（viewport 外挙動、コンテキストメニュー）は宣言的定義のみで制御可能

### NFR-3: 一貫性

- 全タイルタイプで共通操作（ドラッグ、リサイズ、削除、edge 接続）の UX が統一されている
- コンテキストメニューの構造（共通 → セパレータ → タイプ固有）は全タイルで一貫する

## Success Criteria

- [ ] SC-1: 全タイルタイプが共通プロパティ（id, type, geometry, title, zIndex, visible, locked）を持つ
- [ ] SC-2: タイルタイプごとに min/max/default サイズが個別定義され、リサイズ時に制約が適用される
- [ ] SC-3: viewport 外挙動がタイルタイプごとに宣言的に定義され、ターミナル・エディタはアンマウント、メモは keep で動作する
- [ ] SC-4: worktree → session の edge が自動生成され、その他のタイル間で手動 edge が作成可能
- [ ] SC-5: コンテキストメニューが共通項目（削除、edge 接続、サイズリセット）+ タイプ固有項目で構成される
- [ ] SC-6: 新しいタイルタイプを宣言的登録のみで追加できる（共通仕様の変更が不要）
