# 実装計画: 画面表示時刻のシステムロケール変換

**仕様ID**: `SPEC-12af0c9a` | **日付**: 2026-01-09 | **仕様書**: `specs/SPEC-12af0c9a/spec.md`
**入力**: `specs/SPEC-12af0c9a/spec.md` からの機能仕様

## 概要

CLIとWeb UIで表示されるすべての時刻をシステムロケールのタイムゾーンで表示する。既存の表示フォーマットは維持し、保存データやAPIの時刻表現は変更しない。UTC表示の漏れを防ぐため、表示用フォーマッタを各UI層で共通化し、テストで保証する。

## 技術コンテキスト

**言語/バージョン**: TypeScript 5.8  
**主要な依存関係**: Bun, OpenTUI (@opentui/core/@opentui/solid), SolidJS, React 19, Vite, Fastify  
**ストレージ**: ファイル（JSON/JSONL）  
**テスト**: Vitest, Playwright  
**ターゲットプラットフォーム**: macOS, Linux, Windows  
**プロジェクトタイプ**: 単一リポジトリ（CLI + Web UI）  
**パフォーマンス目標**: 既存のレンダリング性能を維持（追加処理は最小限）  
**制約**: 新規依存を追加しない、既存フォーマットを維持する  
**スケール/範囲**: CLI全画面 + Web UI全画面の時刻表示

## 原則チェック

- P2で表示形式の安定性を確保

## テスト戦略

- **ユニットテスト**: CLI/Web UIの時刻フォーマット関数を固定タイムゾーンで検証
- **統合テスト**: 主要画面での表示フォーマット確認
- **エンドツーエンドテスト**: 既存のE2Eに影響が出ないことを確認

## リスクと緩和策

### 技術的リスク

1. **タイムゾーン依存のテストが不安定**  
   - **緩和策**: テスト実行時にタイムゾーンを固定し、決定的なタイムスタンプを使用する

2. **秒/ミリ秒の混在による表示ズレ**  
   - **緩和策**: 既存の型/単位を整理し、表示前に単位を統一する

### 依存関係リスク

1. **Intl実装差による表示差**  
   - **緩和策**: 既存のフォーマット関数を再利用し、必要最小限のロケール依存に留める
