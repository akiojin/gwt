# バグ修正仕様: Version History を最新タグ10件に固定表示する（Issue #1230）

**仕様ID**: `SPEC-91f01230`
**作成日**: 2026-02-25
**更新日**: 2026-02-25
**ステータス**: ドラフト
**カテゴリ**: GUI
**依存仕様**:

- [SPEC-133bf64f](../SPEC-133bf64f/spec.md)（Project Version History 基本仕様）
- [SPEC-c9a2f731](../SPEC-c9a2f731/spec.md)（Version History 即時表示）

**入力**: ユーザー説明: "10件固定でも良いですが、最新の10件にして下さい。"

## 背景

- `Version History` 一覧で古いタグまで表示する仕様変更案があったが、要件は「表示件数は固定でよい。ただし最新10件を表示すること」に確定した。
- 一覧の目的は直近リリースの把握であり、件数を固定した方が視認性と応答性が安定する。

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - 最新タグ10件をすぐ確認できる (優先度: P0)

利用者として、Version History を開いたときに最新タグ10件だけを確認したい。

**独立したテスト**: タグが11件以上あるリポジトリで Version History を開き、最新タグ10件のみ表示されることを確認する。

**受け入れシナリオ**:

1. **前提条件** `v*` タグが 12 件存在する、**操作** `Version History` タブを開く、**期待結果** 一覧にタグが 10 件だけ表示される。
2. **前提条件** `v*` タグが 12 件存在する、**操作** 一覧表示を確認する、**期待結果** `v1.0.12` から `v1.0.3` まで表示され、`v1.0.2` 以前は表示されない。

---

### ユーザーストーリー 2 - Unreleased を一覧から除外できる (優先度: P1)

利用者として、Version History 一覧ではタグだけを見たい。

**独立したテスト**: `list_project_versions` が `unreleased` を返しても UI 一覧に表示されないことを確認する。

**受け入れシナリオ**:

1. **前提条件** API 応答に `unreleased` とタグが含まれる、**操作** `Version History` タブを開く、**期待結果** 一覧にはタグのみが表示される。

## エッジケース

- タグが 0 件の場合は「No version tags found.」を表示する。
- タグが 10 件未満の場合は存在するタグ件数のみ表示する。

## 要件 *(必須)*

### 機能要件

- **FR-001**: `Version History` タブは一覧取得時に `list_project_versions` を `limit=11` で呼び出さなければならない。
- **FR-002**: UI は `id == "unreleased"` の項目を除外し、残りの先頭 10 件のみを表示しなければならない。
- **FR-003**: 各表示タグに対してのみ `get_project_version_history` を呼び出さなければならない。
- **FR-004**: 一覧の順序は `list_project_versions` が返す順序（`v*` バージョン降順）を維持しなければならない。

### 非機能要件

- **NFR-001**: 修正は表示件数制御に限定し、要約生成・キャッシュ機能の既存動作を変更しない。

## 制約と仮定

- 「最新」はタグ作成日時ではなく `v*` のバージョン番号順（`--sort=-v:refname`）を指す。
- `list_project_versions` のバックエンド API 仕様（`limit` は Unreleased を含む件数上限）は維持する。

## 成功基準 *(必須)*

- **SC-001**: タグが 11 件以上あるリポジトリで、一覧表示件数が常に 10 件になる。
- **SC-002**: `Unreleased (HEAD)` が一覧に表示されない。
- **SC-003**: Frontend テストで `limit=11` 呼び出し・10件制限・unreleased 除外が検証される。
