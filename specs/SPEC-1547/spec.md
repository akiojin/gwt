### 背景

gwt の全コンセプト（エージェント、worktree、Issue、PR、タスク）を 2D スプライトとしてトップダウン ¾ ビュー開発スタジオ内に表現する。インタラクションモデルはハイブリッド方式を採用し、スタジオ内オブジェクトのクリックでオーバーレイ UI を表示、コマンドパレット（Cmd+K）でクイックアクションを実行する。

全エンティティは 2D スプライト（ピクセルアート）で表現し、3D モデルは使用しない。1リポジトリ = 1スタジオのモデルを採用する。

スタジオワールドはTilemap＋フリー配置ハイブリッドで構成し、デスク増加に応じて動的に拡張する。描画はPixel Perfect Cameraを使用してピクセルアートの精密な表示を保証する。スタジオには入口（ドア）があり、キャラクターの入退場はドアを起点とする。

### コアエンティティモデル

gwt のエンティティには明確な依存チェーンがある:

```text
Issue（仕事の単位、全ての起点）
  ↓ Issueがあるから
Worktree = デスク（作業場所）が作れる
  ↓ デスクがあるから
Agent (Terminal) = スタッフ (Developer) を雇用できる
```

- **Issue**: 全ての起点。Issueがあるから Worktree が作成でき、デスクが用意される
- **Worktree = デスク**: 作業場所の物理的表現。デスクがあるからスタッフを雇用できる
- **Agent = スタッフ**: スタッフがいる＝エージェントターミナルが存在する
- **スタッフの種類**: Lead（管理者）と Developer（開発者）
- **Lead**: デスクを持たない特別なスタッフ。Issue-Worktree-Agentチェーンの外にいる。スタジオ内を巡回しオーケストレーションを行う。ターミナル（オーケストレーションログ）は存在するがデスクには紐付かない。LeadはAPI直接呼び出しで動作し（CLIエージェントではない）、完全自律で動作する。Agentツールとしてコンテキスト（ファイル内容、Git状態、Issue情報等）を提供される

**例外: Issue なしの Worktree**
- 基本は Issue 起点だが、ユーザーが手動で作った既存 Worktree 等は「Issueなしデスク」として表示される
- 後から Issue を紐付けることも可能

現行 Project Mode のエンティティ:
- **LeadOrb**: 4 状態（idle / thinking / waiting_approval / orchestrating）
- **AgentAvatar**: 開発者スプライト
- **IssuePlot**: Issue 表示
- **TaskBar**: タスク進捗

これらを 2D ピクセルアートスプライト＋アニメーション＋ビジュアルステート表現に進化させる。

**エンティティマッピング（スタジオ内配置）:**

| gwt コンセプト | スタジオ内表現 |
|---------------|--------------|
| Developer（スタッフ） | デスクでPCに向かうキャラクター（2Dピクセルアートスプライト、状態アニメーション付き） |
| Lead（スタッフ） | スタジオ内を巡回する特別なキャラクター（専用デザインの2Dスプライト、デスクなし） |
| Worktree = デスク | デスク＋PC（スタジオ内の家具オブジェクト） |
| Worktree（スタッフなし） | 空席デスク＋PC（キャラクターなし、Agentが起動されていない） |
| リモートブランチ（worktree なし） | 半透明デスク＋PC |
| Issue（未連携） | スタジオ内にランダム浮遊する Issue マーカー「!」（バグ/問題マーカー） |
| Issue（ブランチ連携済み） | 連携先デスク付近に浮遊する「!」マーカー |
| PR | デスクに付随するステータスバッジ（アイコン＋色のみ、PR存在時のみ表示） |
| Task | Issue マーカーのサブアイテム（進捗インジケーター） |
| Agent Terminal | スタッフクリック時に表示されるオーバーレイ UI パネル（スタッフがいる＝ターミナルが存在する） |
| Lead質問 | Lead付近に浮遊する「?」マーカー（緩急度別色分け） |

**エージェントアニメーション（Developerスタッフの2Dスプライトアニメーション）:**

| 状態 | アニメーション表現 |
|------|-------------------|
| running（実行中） | デスクでPCに向かってタイピング |
| waiting_input（入力待ち） | 手を振って注意を引く |
| stopped（停止/休憩） | スタジオ内をランダム歩行＋家具インタラクション（コーヒーマシン→飲むアニメ、本棚→読むアニメ、ホワイトボード→書くアニメ） |
| idle（待機） | デスクに座って待機 |

**エージェント種別:**

| 種別 | 種類 | 動作方式 |
|------|------|---------|
| Claude Code | CLIエージェント | PTY経由 |
| Codex (OpenAI) | CLIエージェント | PTY経由 |
| Gemini (Google) | CLIエージェント | PTY経由 |
| GitHub Copilot | CLIエージェント | PTY経由 |
| ユーザーカスタム | CLIエージェント（CLIパス＋引数指定） | PTY経由 |

**インタラクションパターン:**

ヒット判定優先順位: **Lead > Developerキャラクター > デスク > Issue/? マーカー**（貫通なし）

- **Leadクリック** → Lead のライブターミナル（オーケストレーションログ）をオーバーレイ表示（Leadはデスクなし、スタジオ巡回中にクリック）
- **Developer（キャラクター）クリック** → エージェントが起動しているターミナル（ライブセッション）をオーバーレイ表示（stopped状態でデスクから離れた位置にいても同様。スタッフがいる＝ターミナルが存在する）
- **デスク（Developerスタッフ着席状態）クリック** → デスクコンテキストメニュー表示
- **空席デスククリック** → 空席デスクコンテキストメニュー表示（Hire Agent / Terminal / Git / Delete Worktree）
- **半透明デスククリック** → worktree 作成確認ダイアログ（リモートブランチ名から自動決定、ユーザー承認/編集可）→ スタッフ雇用
- **Issue マーカー「!」クリック** → オーバーレイで Issue 詳細表示 + スタッフ雇用アクション（雇用実行で自動ブランチ連携: Issue → Worktree → デスク → スタッフ）
- **?マーカークリック** → オーバーレイで質問表示 + 回答フォーム（テキスト入力 + Approve/Reject）
- 複雑な入力 → 2D オーバーレイフォーム
- クイックアクション → コマンドパレット（Cmd+K） / ホットキー

### スタジオワールド構造

**レイアウト方式: Tilemap＋フリー配置ハイブリッド**
- 床・壁・通路はTilemapで構成（タイルベース）
- デスク・家具はフリー配置（タイルグリッドにスナップしつつ自由に移動可能）
- A* Pathfinding Pro のGridGraphをTilemap上に生成し、パスファインディングに使用

**動的スタジオ拡張:**
- デスク数が増加するとスタジオが自動的に拡張（壁・床タイルの追加）
- 最小サイズ: 4デスク分（初期状態）
- 拡張単位: 行単位で拡張（2デスク分の行追加）
- 縮小: デスク削除時に不要な行を自動縮小

**スタジオ入口（ドア）:**
- スタジオの下端または側面にドアを配置
- 全キャラクターの入退場はドアを経由
- 入場: ドアから出現 → デスクまで歩行
- 退場: 現在位置 → ドアまで歩行 → ドアで消滅

**1リポジトリ = 1スタジオ:**
- 各リポジトリが1つのスタジオとして表現される
- モノレポ対応: サブディレクトリ単位ではなくリポジトリルート単位

**Pixel Perfect Camera:**
- Unity の Pixel Perfect Camera コンポーネントを使用
- ピクセルアートのサブピクセルずれを防止
- ズーム操作時もピクセル整合性を維持

### デスクコンテキストメニュー

Developerスタッフが着席しているデスク（Worktree + Agent起動中）をクリックすると、
デスク付近にフローティングコンテキストメニューが表示される。

**メニュー構成:**

| 項目 | 説明 | 開くウィンドウ | 利用不可時 |
|------|------|--------------|-----------|
| Terminal | プレーンターミナル（Agent非起動シェル） | TerminalOverlayPanel（worktreeディレクトリで新規PTYセッション起動） | — |
| Summary | AI要約（エージェントセッションのサマリー） | SummaryOverlayPanel（セッションサマリーマークダウン表示） | 未生成時グレーアウト |
| Git | Git詳細情報（diff・コミット・stash） | GitDetailPanel（スナップショット表示＋手動リフレッシュ） | — |
| PR | PRステータス・チェック確認 | PrDetailPanel（ステータス確認＋GitHubリンク。マージ操作は含まない） | PR未作成時グレーアウト |
| Fire Agent | スタッフ解雇 | 確認ダイアログ → 解雇実行 | — |

**ビジュアルデザイン:**
- ピクセルアート統一: ドット絵フォント＋ドットボーダーで描画
- RPGの選択メニュー風の外観
- 利用不可項目はグレーアウト表示（クリック不可）

**メニュー表示位置:**
- Screen Space に描画（ズームに影響されない固定サイズ）
- デスクのスクリーン座標付近に表示
- 画面端にある場合は自動クランプ（画面外にはみ出さない）

**ターミナルの区別:**
- スタッフクリック → そのスタッフのエージェントターミナル（PTYセッション共有、ライブ出力表示）。スタッフがいる＝ターミナルが存在する
- デスクコンテキストメニュー → Terminal → プレーンターミナル（新規PTYセッション、worktreeディレクトリで起動。エージェントとは無関係のシンプルなシェル）

**プレーンターミナルのPTYライフサイクル:**
- パネルを閉じる = PTYセッション即座終了（新規セッションは毎回作成）
- バックグラウンド維持なし

**Fire Agent フロー（状態別段階的警告）:**

| エージェント状態 | 確認ダイアログ |
|----------------|--------------|
| idle / stopped | シンプル確認: "Fire this agent? Agent will stop and leave the desk." [Cancel] [Fire] |
| running | 追加警告: "⚠ Agent is working! Unsaved changes may be lost. Force fire?" [Cancel] [Force Fire] |
| waiting_input | 選択肢提示: "Agent is waiting for input. Respond first or force fire?" [Cancel] [Respond] [Force Fire] |

1. 確認 → エージェントプロセス停止（SIGTERM → graceful shutdown 待機 → 必要に応じ SIGKILL）
2. デスク → 即座に空席状態に変化（再雇用可能）
3. 退場アニメーション（キャラクター → ドアへ歩行 → ドアで消滅）は装飾的に並行再生

**Fire Agent ≠ Worktree 削除:**
- Fire Agent: エージェントを停止し、デスクを空席にする操作。Worktreeは維持される
- Worktree 削除: 空席デスクコンテキストメニューの Delete Worktree で実行する別操作。デスクが消滅する

**表示条件:**
- スタッフ着席デスクのみ表示（空席デスク・半透明デスクでは表示しない）
- メニュー外クリックで閉じる
- ESCキーで閉じる
- メニュー項目選択後、メニューは自動で閉じる

### 空席デスクコンテキストメニュー

Worktreeはあるがエージェントが起動していない空席デスクをクリックすると、空席デスク専用のコンテキストメニューが表示される。

**メニュー構成:**

| 項目 | 説明 | アクション |
|------|------|-----------|
| Hire Agent | スタッフ雇用 | 雇用UIオーバーレイ表示（プロファイル選択＋タスク指定） |
| Terminal | プレーンターミナル | TerminalOverlayPanel（worktreeディレクトリで新規PTYセッション起動） |
| Git | Git詳細情報 | GitDetailPanel（diff・コミット・stash） |
| Delete Worktree | Worktree削除 | 確認ダイアログ → Worktree削除 → デスク消滅 |

**Delete Worktree フロー:**
1. Delete Worktree 選択 → 確認ダイアログ表示（"Delete this worktree? Branch will be preserved on remote."）
2. 未コミット変更がある場合は追加警告: "⚠ Uncommitted changes will be lost!"
3. 確認 → Worktree削除 → デスクスプライトフェードアウト消滅
4. 紐付きIssue「!」マーカーがある場合は未連携状態に戻り、自由浮遊を再開

**表示条件:**
- 空席デスク（Worktree あり + Agent未起動）のみ表示
- スタッフ着席デスクでは表示しない（スタッフ着席時はデスクコンテキストメニューを使用）
- メニュー外クリック / ESCキーで閉じる

### 家具インタラクション

スタジオ内にはデスク以外の家具オブジェクトが配置され、stopped状態のDeveloperがインタラクションする。

**家具種類とアニメーション:**

| 家具 | インタラクション | アニメーション |
|------|----------------|--------------|
| コーヒーマシン | 飲む | キャラクターがコーヒーマシンの前に立ち、カップを持って飲むアニメーション |
| 本棚 | 読む | キャラクターが本棚の前で本を読むアニメーション |
| ホワイトボード | 書く | キャラクターがホワイトボードに書き込むアニメーション |

**動作フロー:**
1. stopped状態のDeveloperがランダムに家具を選択
2. 家具まで歩行（A* パスファインディング）
3. 家具の前でインタラクションアニメーション再生（3〜8秒）
4. 完了後、次の家具またはランダム歩行を選択
5. タスク受信時は即座にデスクへ帰還

**家具配置:**
- 家具はスタジオ内に固定配置（ユーザーによるドラッグ移動は不可）
- スタジオ拡張時に家具も自動追加

### キャラクター命名規則

- 完全にランダムな人名を自動生成（性別・国籍問わず）
- エージェント種別（Claude Code / Codex / Gemini等）は名前ではなくラベルで表示
- 名前はキャラクター上部のNameLabelに表示
- エージェント種別はNameLabel下部またはツールチップに小さく表示

**エージェント種別ラベル:**

| エージェント種別 | ラベル表示 |
|-----------------|-----------|
| Claude Code | Claude Code |
| Codex (OpenAI) | Codex |
| Gemini (Google) | Gemini |
| GitHub Copilot | Copilot |
| ユーザーカスタム | Custom |

**表示例:**
```text
   [Alice]         ← ランダム生成名
  Claude Code      ← エージェント種別ラベル
   (キャラクタースプライト)
```

### デスクドラッグ

**ドラッグ操作:**
- デスクをマウスドラッグで移動可能
- ドラッグ中はデスク＋着席キャラクター（いる場合）が一緒に移動
- タイルグリッドにスナップ（ドロップ時にグリッド位置に合わせる）
- 他のデスクや家具との重なり防止（配置不可位置ではゴーストをグレー表示）

**A* グラフ再計算:**
- ドロップ確定時にA* Pathfinding ProのGridGraphをバッチ再計算
- ドラッグ中はグラフ再計算しない（パフォーマンス考慮）
- 再計算範囲: 移動元＋移動先の周辺タイルのみ（部分更新）

### Lead質問マーカー「?」

Leadがオーケストレーション中にユーザーの判断を必要とする場合、スタジオ内に「?」マーカーが出現する。
Issue「!」マーカーと対になるシステムで、プロジェクト全体の判断要求を視覚的に表現する。

**?マーカーとwaiting_inputの区別:**
- **?マーカー**: Lead独自のプロジェクト判断要求（「このIssueをどう扱うか？」「プランBで進めて良いか？」等）
- **waiting_input**: Developer個別のタスク内入力待ち（手振りアニメーションで表現）
- 両者は完全に独立したシステム

**ライフサイクル:**
1. Lead が判断要求を生成
2. 「?」マーカーが Lead 付近に浮遊開始
3. ユーザーが「?」をクリック
4. オーバーレイパネルで質問内容表示＋回答フォーム（テキスト入力 + Approve/Reject ボタン）
5. 回答送信後、「?」マーカー消滅

**緩急度による色分け:**

| 緩急度 | 色 | アニメーション |
|--------|-----|---------------|
| ブロッキング（承認必須、Developer待機中） | 赤 | 点滅＋拡大アニメ |
| 通常 | 黄 | 穏やかな浮遊 |
| 低優先度 | 青 | 微光浮遊 |

**配置:** Lead 付近に浮遊（Lead が移動すると追従）

### Issueマーカー「!」とブランチ連携

**Issue-Branch連携（現行gwt既存機能）:**
- IssueマーカーはGitブランチと連携する。連携状態によって「!」マーカーの位置と挙動が変わる
- **未連携Issue**: ブランチが紐付いていないIssue。スタジオ内を自由に浮遊
- **ブランチ連携済みIssue**: ブランチ（=Worktree=デスク）に紐付いたIssue。デスク付近に浮遊固定

**連携タイミング: スタッフ雇用時に自動連携**
1. ユーザーが「!」マーカーをクリック → Issue詳細パネル表示
2. パネル内でスタッフ雇用を実行
3. Issue番号からブランチ名を自動生成（例: `feature/issue-123`）
4. ブランチ作成 → Worktree作成 → デスク出現
5. 「!」マーカーがデスクへ飛行アニメーション → デスク付近に浮遊固定
6. スタッフがドアから入場 → デスクまで歩行 → デスク着席

**浮遊＋クラスタリング:**
- 未連携Issueはスタジオ内を緩やかに浮遊
- 類似ラベルのIssueは近くに集まるクラスタリング
- デスクやキャラクターとの重なり回避ロジック

**ライフサイクル:**
1. `[open, 未連携]` → スタジオ内自由浮遊
2. `[open, ブランチ連携]` → 「!」マーカーがデスクへ飛行 → デスク付近に浮遊固定
3. `[in_progress]` → デスク付近に浮遊継続
4. `[closed]` → フェードアウト消滅

### PRステータスバッジ

- PR存在時のみデスクに小さなバッジを表示
- アイコン＋色のみ: ○=open(緑), ✔=merged(紫), ×=closed(赤), CI状態で色変化（緑=pass, 赤=fail）
- ホバーで詳細ツールチップ（PR番号、タイトル、CI状態、レビュー状況）
- クリックでコンテキストメニューのPR項目と同じPrDetailPanelを開く

### AI要約（Summary）

**イベント駆動生成:**
要約はバックグラウンドでイベント駆動により生成・キャッシュされる。パネルを開いた時はキャッシュを即座表示。

| トリガーイベント | 要約更新内容 |
|----------------|-------------|
| agent running → idle（タスク完了） | 完了タスクのサマリー |
| git commit 検出 | コミット内容の要約 |
| agent waiting_input | 入力待ちの内容と文脈 |
| 5分以上 running 継続 | 現在の作業進捗サマリー |

**吹き出しとSummaryパネルの関係:**
- 吹き出し = テンプレート固定の短文（"Working on #123...", "Waiting for input", "On break"）。AI要約切替は行わない
- Summaryパネル = AI生成の詳細マークダウンレポート

### Gitパネル

- パネルを開いた時点でのスナップショットを表示（diff、コミット履歴、stash）
- リフレッシュボタンで手動更新
- リアルタイム更新は行わない（Agent running中でも自動更新しない）

### スタッフ雇用UI

**空席デスクコンテキストメニュー → Hire Agent 選択時:**
- オーバーレイフォームでスタッフ雇用を実行
- 基本はLeadが自動的に雇用を行うが、ユーザーが直接雇用も可能
- **入力項目:**
  - プロファイル選択（Claude Code / Codex / Gemini / GitHub Copilot / ユーザーカスタムから選択）
  - ユーザーカスタム選択時: CLIパス＋引数を指定
  - タスク指定（Issue選択 or 自由テキスト）

**Issue「!」マーカーからの雇用時:**
- Issue詳細パネル内でスタッフ雇用を実行
- 雇用実行で自動的にブランチ作成 + Worktree作成 + デスク出現 + Issue連携
- ブランチ名はIssue番号から自動生成（例: `feature/issue-123`）
- 依存チェーン全体を一括実行: Issue → Worktree(デスク) → Agent(スタッフ)

**半透明デスク（リモートブランチのみ）クリック時:**
- リモートブランチ名からworktreeパスを自動生成
- 確認ダイアログでユーザーが承認または編集
- 承認後、worktree作成 → スタッフ雇用の一括実行

### Issue起点のデスク出現フロー

デスクの配置はIssueベースで駆動される（推奨フロー）。依存チェーン全体を一括で実行:

```text
Issue #123 (スタジオ内浮遊「!」マーカー)
  ↓ ユーザーが「!」クリック → Issue詳細パネル → スタッフ雇用実行
自動: ブランチ作成 (feature/issue-123)
  ↓
自動: Worktree 作成 = デスク出現
  ↓
「!」マーカーがデスクへ飛行 → デスク付近に浮遊固定
  ↓
スタッフ(Developer)がスタジオ入口（ドア）から入場 → デスクまで歩行 → デスク着席 → タスク開始
```

**例外: Issueなしデスク**
- ユーザーが手動で作成した既存Worktree等は、Issueなしでもデスクとして表示される
- 後からIssueを紐付けることも可能

### アセット方針

- 全キャラクター・オブジェクトは 2D ピクセルアートスプライトで統一（16x16 ベース）
- 3D モデルは一切不要
- **キャラクタースプライト**: LimeZu Modern Interiors のキャラジェネレータを使用（100+衣装、200髪型、80アクセサリ、9肌色）
  - エージェント種別×ジョブタイプの見た目バリエーションを LimeZu キャラジェネレータで生成
  - キャラクター外見はプロファイルごとに固定（同じモデルは常に同じ外見）
  - idle/run/read/lift 等のアニメーションはアセットに付属
- **追加アニメーション**: BoneToPix ($27) で 3D→ドット絵アニメ自動変換（大量バリエーション対応）
- **Aseprite パイプライン**: talecrafter/AnimationImporter (無料) で .ase ファイル直接インポート
- **サウンドエフェクト**: ピクセルアート風SE（メニュー開閉、雇用、解雇、通知、エラー等）。事前アセット（.wav/.ogg）のみ使用し、ランタイム生成は行わない
- **家具スプライト**: LimeZu Modern Interiors のオフィス家具セット（コーヒーマシン、本棚、ホワイトボード等）
- 詳細は `docs/pixelart.md` を参照

### キャラクター状態遷移

Developer スタッフの状態遷移:

```text
[雇用] → ドアから入場 → デスクまで歩行 → デスク着席 → idle
idle → [タスク受信] → running (タイピング)
running → [入力待ち] → waiting_input (手振り + 吹き出し)
waiting_input → [入力受領] → running
running → [タスク完了] → idle
idle → [一定時間経過] → stopped (スタジオ内ランダム歩行＋家具インタラクション: コーヒーマシン→飲む、本棚→読む、ホワイトボード→書く)
stopped → [タスク受信] → デスクへ歩行 → running
任意状態 → [解雇] → プロセス停止 → デスク即座空席化 → ドアへ歩行退場アニメ並行再生 → ドアで[消滅]
```

Lead 固有の動作（デスクなし、スタジオ巡回、完全自律）:

```text
[プロジェクト開始] → ドアから入場 → スタジオ中央へ移動
idle → スタジオ内巡回（ランダム、デスク間を歩行）
[指示送信] → 対象デスクへ移動 → 指示アニメーション → 巡回に戻る
[Issue 判断] → Issue マーカーへ移動 → 判断アニメーション
[判断要求] → ?マーカー生成 → Lead付近に浮遊 → ユーザー回答待ち
```

### 移動システム

- パスファインディング: A* Pathfinding Pro（GridGraph、Tilemap連動）
- 移動速度: PPU (Pixels Per Unit) ベースで定義
- 衝突回避: キャラクター同士の重なり防止
- 歩行アニメーション: 4方向（LimeZu アセット標準: 上・下・左・右）
- デスクドラッグ: デスク＋着席キャラクター連動移動、ドロップ時にグリッドスナップ＋A* GridGraph部分再計算（バッチ）

### カメラ操作

- パン: マウスドラッグまたはWASD/矢印キーでスタジオ内を移動
- ズーム: マウスホイールまたはショートカットキーで拡大・縮小
- Pixel Perfect Camera: ズーム操作時もピクセル整合性を維持
- 大規模スタジオ（50+デスク）に対応

### 吹き出し表示

- 表示条件:
  - Lead 巡回時: Developer デスクに近づくと自動表示
  - Developer 状態変化時: waiting_input → 手振り + 吹き出し
  - ユーザーホバー時: キャラクターにマウスオーバーで表示
- 表示時間: 5秒（自動消去）、クリックで即時消去
- テキスト生成元: テンプレート固定のみ（AI要約切替は行わない）
  - "Working on #123..."
  - "Waiting for input"
  - "On break"

### オーバーレイパネル管理

- 複数パネルの同時表示が可能（Terminal + Git 等）
- 各パネルはドラッグで移動可能
- ESCキーでアクティブパネルを閉じる
- エンティティ別独立パネル（統一タブパネルではない）

### キーボードショートカット

全操作にキーボードショートカットを割り当てる:
- ESC: パネル/メニューを閉じる
- Cmd+K: コマンドパレット
- Cmd+T: Terminal
- Cmd+G: Git
- スペース: カメラリセット
- 数字キー: コンテキストメニュー項目選択
- 等（詳細はUX設計時に確定）

### エラー表現

- **トースト通知**: 画面上部/下部に即時表示、5秒で自動消去。エラー内容を簡潔に表示
- **エンティティ状態反映**: エラーが発生したエンティティ（デスク、スタッフ）に赤いエラーアイコンを持続表示。クリックで詳細確認

### メモリ管理

- ObjectPool パターン適用対象:
  - 吹き出し UI (BubblePool)
  - Issue マーカー (IssueMarkerPool)
  - ?マーカー (QuestionMarkerPool)
  - エフェクト (EffectPool)
- プールサイズ: 初期16、最大64（自動拡張）
