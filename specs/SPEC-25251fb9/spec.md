# バグ修正: ターミナル行数増加時のトラックパッドスクロール不能

**仕様ID**: `SPEC-25251fb9`
**作成日**: 2026-02-24
**更新日**: 2026-02-24
**ステータス**: ドラフト
**カテゴリ**: GUI
**GitHub Issue**: #1209

## 背景

- ターミナル出力の行数が増えると、トラックパッドによるスクロールが効かなくなる
- 現在の実装は `viewport.scrollTop` を直接操作しているが、xterm.js の内部ビューポート状態の同期が scroll イベント経由（非同期）で行われる
- ターミナル出力が高頻度で届く場合、xterm.js が古い `viewportY` に基づいてオートスクロールし、手動スクロールを上書きする競合が発生する
- トラックパッド / マウスホイール判定ロジックも脆弱で、過去に複数回の修正を重ねている

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - 通常バッファでのトラックパッドスクロール (優先度: P0)

ユーザーとして、ターミナル出力が多い状態でもトラックパッドで確実にスクロールしたい

**受け入れシナリオ**:

1. **前提条件** ターミナルが通常バッファモード（alternate buffer でない）、**操作** トラックパッドでスクロール、**期待結果** `terminal.scrollLines()` が呼ばれスクロールが実行される
2. **前提条件** 通常バッファでスクロール済み、**操作** ターミナルに新規出力到着、**期待結果** ユーザーのスクロール位置が維持される（オートスクロールで上書きされない）

### ユーザーストーリー 2 - alternate buffer でのホイール委譲 (優先度: P0)

ユーザーとして、vim や tmux 等のフルスクリーンアプリ実行中はホイールイベントがアプリに届いてほしい

**受け入れシナリオ**:

1. **前提条件** ターミナルが alternate buffer モード、**操作** ホイールイベント発生、**期待結果** イベントが preventDefault / stopImmediatePropagation されず xterm.js に委譲される

### ユーザーストーリー 3 - サブライン蓄積によるスムーズスクロール (優先度: P1)

ユーザーとして、小さなトラックパッドデルタでも蓄積されて確実にスクロールしたい

**受け入れシナリオ**:

1. **前提条件** 通常バッファ、**操作** 小さな deltaY（1行未満）を複数回送信、**期待結果** ライン単位に蓄積され、閾値を超えた時点で scrollLines が呼ばれる

## エッジケース

- viewport 要素が未生成の場合は何もしない
- scrollHeight == clientHeight（スクロール不要）の場合は scrollLines を呼ばない
- deltaY === 0 && deltaX === 0 の場合は早期リターン
- 軸切り替え時（vertical ↔ horizontal）にリメインダーがリセットされる

## 要件 *(必須)*

### 機能要件

- **FR-001**: 通常バッファでのホイールイベントは `terminal.scrollLines()` API でスクロールする（viewport.scrollTop 直接操作を廃止）
- **FR-002**: alternate buffer（type === "alternate"）では xterm.js にホイール処理を委譲する
- **FR-003**: ピクセルデルタをライン単位に変換し、サブラインのリメインダーを蓄積する
- **FR-004**: トラックパッド / マウスホイール判定ロジック（`isTrackpadLikeWheel`）を削除し、通常バッファでは一律カスタムハンドリングとする

### 非機能要件

- **NFR-001**: 既存のホイールスクロールテストが新しい挙動に合わせて更新される

## 制約と仮定

- xterm.js v6 の `terminal.scrollLines()` と `terminal.buffer.active.viewportY` が公開 API として利用可能
- alternate buffer 判定に `terminal.buffer.active.type` を使用

## 成功基準 *(必須)*

- **SC-001**: 通常バッファでホイールイベントが `terminal.scrollLines()` 経由で処理される
- **SC-002**: alternate buffer ではイベントが xterm.js に委譲される
- **SC-003**: 全既存テスト + 新規テストが通過する
