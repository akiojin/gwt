### 背景

gwt は Rust/Tauri 版から Unity/C# 版へ完全移行する。
既存ユーザーの ~/.gwt/ データ（設定、セッション、プロファイル）を Unity 版で自動的に読み取り、必要に応じて形式変換する。

#### マイグレーション方式

- **完全自動**（ユーザー操作不要、初回起動時に自動実行）
- マイグレーション失敗時は**スキップ**して新規状態で起動（エラーでブロックしない）
- 失敗理由はログに記録し、ユーザーには簡潔な通知のみ表示
- **元ファイルは保持**（削除しない）。変換後もTOML元ファイルはそのまま残す
- バックアップは実行するが、ロールバックUIは提供しない（手動復元用）

#### TOML→JSON変換対象

以下の全TOMLファイルをJSON形式に変換する:
- `~/.gwt/config.toml` → JSON形式
- `~/.gwt/config/profiles.toml` → JSON形式
- `~/.gwt/config/agent-config.toml` → JSON形式
- `~/.gwt/agents.toml` → JSON形式
- その他 `~/.gwt/` 配下に存在するTOMLファイル全般

### ユーザーシナリオ

| ID | シナリオ | 優先度 |
|----|---------|--------|
| US-1 | Rust版を使用していたユーザーがUnity版を初回起動すると、自動的にデータが移行される | P0 |
| US-2 | 移行後、以前のプロジェクト・セッション・設定がそのまま利用可能 | P0 |
| US-3 | 移行前のデータがバックアップされており、問題時に手動復元可能 | P1 |

#### US-1 詳細

1. Unity版を初回起動する
2. ~/.gwt/ に既存データが検出される
3. データ形式のバージョンチェックが行われる
4. 自動マイグレーションが開始される（プログレスバー表示）
5. 移行完了後、プロジェクト選択画面に移行済みプロジェクトが表示される
6. 元のTOMLファイルはそのまま保持される（削除しない）
