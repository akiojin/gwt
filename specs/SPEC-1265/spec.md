# バグ修正仕様: Windows版 Launch Agent で `npx.cmd` 起動失敗を防止する

**仕様ID**: `SPEC-1265`
**作成日**: 2026-02-26
**更新日**: 2026-02-26
**ステータス**: ドラフト
**カテゴリ**: Core / Terminal
**依存仕様**:

- SPEC-e2d4f801（OS環境変数の自動継承）

**入力**: ユーザー説明: "Issue #1265を解決して"

## 背景

- Windows 環境で Launch Agent 実行時に `npx.cmd` が `内部コマンドまたは外部コマンドとして認識されない` エラーになり起動できない。
- 失敗ログには ` '\"C:\Program Files\nodejs\npx.cmd\"' ` のような外側クォート混入が見られ、実行トークン解釈が崩れている。
- 既存の `.cmd/.bat` 判定と Windows 実行式組み立てはクォート混入ケースを想定しておらず、再発防止には共通正規化が必要。

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - Windows で Launch Agent を正常起動したい (優先度: P0)

Windows ユーザーとして、Launch Agent を実行したときに `npx` フォールバック経路でもエラーなくエージェントを起動したい。

**独立したテスト**: Windows 実行コマンド解決ロジックに外側クォート混入パスを与え、`cmd.exe /C` で実行可能な式が生成されること。

**受け入れシナリオ**:

1. **前提条件** `command` が ` '\"C:\Program Files\nodejs\npx.cmd\"' `、**操作** Launch Agent の Windows コマンド解決を実行、**期待結果** `cmd.exe /C "C:\Program Files\nodejs\npx.cmd" --yes ...` 形式で起動される
2. **前提条件** `command` が `"C:\Program Files\nodejs\npx.cmd"`、**操作** 同上、**期待結果** 正規化後に `.cmd` として判定され起動に成功する

---

### ユーザーストーリー 2 - 既存挙動を壊したくない (優先度: P1)

開発者として、通常の `codex` やクォートなし `npx.cmd` など既存ケースの起動挙動を維持したい。

**独立したテスト**: 既存の `resolve_spawn_command_*` テストが引き続き全て通ること。

**受け入れシナリオ**:

1. **前提条件** `command` が `C:\Tools\npx.cmd`、**操作** Windows 解決を実行、**期待結果** 既存同様 `cmd.exe /C ...` が返る
2. **前提条件** `command` が `codex`、**操作** Windows 解決を実行、**期待結果** 既存同様 PowerShell wrapper 経路が維持される

## エッジケース

- 外側クォートが多重（`'"..."'` など）でも、外側の対称クォートのみを再帰的に除去する。
- 実行ファイル名に空白を含むケースでも、最終的な `cmd.exe` 引数で必要なダブルクォートが保持される。
- 非バッチコマンド（例: `codex`）では既存の PowerShell wrapper 判定を維持する。

## 要件 *(必須)*

### 機能要件

- **FR-001**: Windows の起動コマンド解決前に、実行コマンド文字列の外側クォート（`'` / `"`）を再帰的に正規化しなければならない。
- **FR-002**: `.cmd/.bat` 判定は正規化後のコマンド文字列で評価しなければならない。
- **FR-003**: `cmd.exe /C` 実行式生成は正規化後のコマンドを使用し、空白を含むパスでも実行可能なトークン列を生成しなければならない。
- **FR-004**: 既存のクォートなしケースおよび非バッチコマンドケースの分岐挙動を維持しなければならない。

### 非機能要件

- **NFR-001**: 変更は `gwt-core` の Windows PTY コマンド解決ロジックに閉じ、公開 API や Tauri command シグネチャを変更しない。
- **NFR-002**: 再現ケースのユニットテストを追加し、関連テストを通過させる。

## 制約と仮定

- 修正対象は Windows のコマンド解決・起動式生成ロジックに限定する。
- Issue #1265 の失敗は実行コマンドトークンのクォート混入が主要因であると仮定する。

## 成功基準 *(必須)*

- **SC-001**: 外側クォート混入の `npx.cmd` 入力で `cmd.exe /C` 経路へ正しく分岐し、実行可能な式が生成される。
- **SC-002**: 既存の Windows コマンド解決テストが回帰なしで通過する。
- **SC-003**: 追加した再現ケースのユニットテストが通過する。
