### 背景

gwt をカイロソフト「ゲーム発展途上国」に着想を得たトップダウン ¾ ビュー開発スタジオとして再構築する。プロジェクト（Git リポジトリ）を開くと、開発スタジオがトップダウン ¾ ビューで展開され、ブランチ・worktree・エージェント・Issue・PR の状態が視覚的に表現される。

ビューはカイロソフト「ゲーム発展途上国」と同一のトップダウン ¾ ビュー（RPG Maker 風）を採用する。真のアイソメトリック（45度回転）ではなく、LimeZu ピクセルアートアセットと互換性のある矩形タイルベースのレイアウトとする。

- Unity URP を 2D ライティングエフェクト用に使用（3D レンダリングではない）
- 現行 Project Mode のコンセプト（LeadOrb, AgentAvatar, PixelSprite のピクセルスプライト表現）をトップダウン ¾ ビュースタジオに進化させる
- カメラはトップダウン ¾ ビュー視点（マウスドラッグによるパンのみ、**ズーム機能なし**）
- 全キャラクターは 2D スプライト（ピクセルアート）で表現
- **キャラクタースプライトは LimeZu アセットをそのまま使用する**（カスタムスプライト作成不要）

#### スタジオレイアウト

| gwt コンセプト | スタジオ内表現 |
|---------------|--------------|
| Worktree（ローカル） | デスク＋PC |
| リモートブランチ（worktree なし） | 半透明デスク＋PC |
| Agent | デスクに座るキャラクター（2Dスプライト） |
| Lead | スタジオ内に常駐するキャラクター（専用デザイン） |
| Issue | スタジオ内にランダム浮遊する Issue マーカー（!マーク等） |
| PR | デスクに付随するステータスバッジ |

#### デスクレイアウト（Issue マーカー位置優先方式）

従来のブランチツリー反映配置を廃止し、Issue マーカーの位置を基準にデスクを配置する方式に変更:

**Issue マーカー:**

- **Issue = バグ/問題マーカー**: スタジオ内にランダムに浮遊する !マーク等のマーカーとして表現
- **100+ の Issue がある場合も全件表示する**（重なり回避アルゴリズムで対応）
- Issue マーカーをクリック → オーバーレイ UI で Issue 詳細を確認
- オーバーレイ UI 内で「エージェント雇用」（worktree 作成 + エージェント起動）を実行可能

**Issue マーカーとデスクの連動:**

- エージェント雇用を実行すると、**Issue マーカーの位置にデスクが出現**し、エージェントが着席する
- Issue マーカーはデスク横に残る（Issue が open の間）
- Issue close → Lead が判断 → マーカー + デスク 両方消滅（worktree 削除）

**Issue 出現タイミング:**

- スタジオ起動時に全 open Issue を表示
- GitHub API ポーリングでリアルタイムに新規 Issue を追加表示

#### ローカル/リモートブランチ表示

- **ローカル worktree = 通常デスク**: 不透明な通常表示のデスク
- **リモートブランチ（worktree なし） = 半透明デスク**: リモートにブランチが存在するがローカルに worktree がない場合、半透明のデスクとして表示
- **半透明デスクをクリック → worktree 作成 + エージェント雇用**: 一気に実行可能

#### main/develop ブランチ

- main/develop は普通のデスクと同じ表示（特別扱いしない）

#### 初期状態（新規プロジェクト）

- Lead キャラクター + main デスク + develop デスクのみが表示される

#### 既存 worktree

- スタジオ起動時に既存の worktree を自動検出し、デスクとして配置する
- Issue に紐づいていない worktree もデスクとして表示される

#### Issue なし worktree 作成

- Lead への指示（チャット/コマンド）で Issue なしの worktree を作成可能
- HUD ボタンからも worktree 作成が可能

#### CI/CD 表示

- CI/CD ステータスは Worktree（デスク）単位で表示する
- CI 失敗デスクは暗く/赤く変化する

#### 動的変化

- Worktree が追加されると新しいデスクがスタジオに出現する
- CI 失敗 → 該当デスクが暗く/赤くなる
- 全テスト Green → スタジオが明るくなる
- Lead は常にスタジオ内に存在（ミーティングルームの概念なし）

#### アプリケーションフロー

1. アプリ起動 → プロジェクト選択画面
2. プロジェクト選択 → シンプルなプログレスバーで初期化
3. 初期化完了 → トップダウン ¾ ビュー開発スタジオが表示される

#### インタビュー確定事項

**スタジオ拡張方向:**
- スタジオの動的拡張は**下方向**に行う
- 新しいデスク行は既存スタジオの下端に追加される
- スタジオ入口（ドア）は上端または側面に配置（下方向拡張と干渉しない位置）
- カメラのパン操作で拡張領域を確認可能

**スタジオ縮小ロジック:**
- **全デスク空席の行のみ縮小対象**とする（1台でもデスクが残っている行は縮小しない）

### 機能要件

- **FR-001**: プロジェクト（Git リポジトリ）を開くと、トップダウン ¾ ビュー開発スタジオを生成する
- **FR-002**: URP を 2D ライティングエフェクト用に使用する（雰囲気変化の表現）
- **FR-003**: トップダウン ¾ ビューカメラシステム（マウスドラッグによるパン、境界制限）を実装する。**ズーム機能は不要（パンのみ）**
- **FR-004**: Issue マーカーをスタジオ内にランダムに浮遊配置する（起動時の全 open Issue + ポーリングによるリアルタイム追加）。**100+ Issue でも全件表示し、重なり回避アルゴリズムで対応する**
- **FR-005**: Issue マーカーのクリックでオーバーレイ UI に Issue 詳細を表示する
- **FR-006**: Issue オーバーレイ UI 内で「エージェント雇用」（worktree 作成 + エージェント起動）を実行する機能を提供する
- **FR-007**: エージェント雇用時に Issue マーカーの位置にデスクを出現させ、エージェントを着席させる
- **FR-008**: Issue close 時に Lead が判断し、マーカー + デスクを消滅（worktree 削除）する
- **FR-009**: ローカル worktree を通常（不透明）デスクとして表示する
- **FR-010**: リモートブランチ（worktree なし）を半透明デスクとして表示する
- **FR-011**: 半透明デスクのクリックで worktree 作成 + エージェント雇用を一括実行する
- **FR-012**: main/develop ブランチを通常デスクとして表示する（特別扱いなし）
- **FR-013**: 新規プロジェクト初期状態で Lead + main + develop デスクのみ表示する
- **FR-014**: スタジオ起動時に既存 worktree を自動検出しデスクとして配置する（Issue 紐づけなしでも表示）
- **FR-015**: Lead への指示および HUD ボタンから Issue なしの worktree 作成を可能にする
- **FR-016**: CI/CD ステータスをデスク単位で表示し、CI 失敗デスクを暗く/赤く変化させる
- **FR-017**: 雰囲気ライティングシステム: 全テスト Green 時にスタジオ全体が明るくなる
- **FR-018**: プロジェクト未選択時は選択UIを表示し、選択後にプログレスバー→スタジオ表示トランジションを実行する
- **FR-019**: スタジオの状態をリポジトリの変更に応じてリアルタイム更新する
- **FR-020**: GitHub API ポーリングによる Issue のリアルタイム追加表示
- **FR-021**: VContainer で IWorldService として DI 登録する
- **FR-022**: **グリッドベース自前実装によるパスファインディングを採用する**（デスク間の直線/グリッド経路移動）。A* Pathfinding Pro は不採用
- **FR-023**: スタジオの動的拡張は下方向に行い、新しいデスク行は既存スタジオの下端に追加する。スタジオ入口（ドア）は上端または側面に配置する

### 非機能要件

- **NFR-001**: worktree 数やエージェント数に上限を設けず、パフォーマンスを維持する
- **NFR-002**: **スタジオ生成（プロジェクト開閉）は5秒以内に完了する**
- **NFR-003**: 60fps をターゲットフレームレートとする
- **NFR-004**: スタジオシーンは Additive Scene として管理する (`SceneManager.LoadSceneAsync(sceneName, LoadSceneMode.Additive)`)
- **NFR-005**: スタジオ切替時は現行シーンをアンロード→新シーンをロード、PTY プロセスは維持
- **NFR-006**: カイロソフト風スタジオでは複雑な障害物回避は不要であるため、**パスファインディングはグリッドベース自前実装**の直線/グリッド経路移動で十分とする
