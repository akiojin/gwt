### 技術コンテキスト

- Unity URP プロジェクトとしてトップダウン ¾ ビュー開発スタジオ基盤を構築
- LimeZu ピクセルアートアセットと互換性のある矩形タイルベースのレイアウト
- VContainer による DI 統合、Additive Scene 管理
- グリッドベース自前パスファインディング（A* Pathfinding Pro 不採用）

### 実装アプローチ

- スタジオ生成ロジックは `IWorldService` インターフェースに集約し、リポジトリ情報の取得とスタジオ内オブジェクト配置を疎結合にする
- **Issue マーカー位置優先の配置**: Issue マーカーの浮遊位置を基準にデスク出現位置を決定する。ブランチツリーベースのレイアウトは廃止
- カメラシステムはトップダウン ¾ ビュー固定角度で、マウスドラッグによるパンのみサポート（**ズーム機能なし**）
- **ローカル worktree = 通常デスク、リモートブランチ = 半透明デスク** の二種表現
- 雰囲気変化は URP の 2D ライティング機能を使用し、CI 状態に応じてライト色・強度を変更する
- 全キャラクターは 2D スプライト（ピクセルアート）で統一し、3D モデルは使用しない。**キャラクタースプライトは LimeZu アセットをそのまま使用**
- **GitHub API ポーリング**で Issue をリアルタイムに取得し、マーカーとして追加表示する。**100+ Issue も全件表示し、重なり回避アルゴリズムで対応**
- **スタジオシーンは Additive Scene として管理する**。スタジオ切替時は現行シーンをアンロード→新シーンをロード、PTY プロセスは維持する
- **パスファインディングはグリッドベース自前実装**を採用し、A* Pathfinding Pro は使用しない
- **スタジオ縮小は全デスク空席行のみ対象**とする

### フェーズ分割

- **Phase S（Setup）**: Unity URP プロジェクトセットアップ、基本シーン構成、VContainer 導入、IWorldService 定義
- **Phase F（Foundation）**: カメラシステム、グリッドベースパスファインディング、アセット調達、Additive Scene 管理
- **Phase U（User story）**: Issue マーカーシステム、デスク表示（通常/半透明）、エージェント雇用フロー、CI/CD 表示、雰囲気ライティング、プロジェクト選択 UI、リアルタイム更新、スタジオ拡張/縮小
- **Phase FIN（Finalization）**: パフォーマンステスト、受け入れテスト

#### リスク

- 大量 Issue マーカーのスタジオ内配置 → **100+ Issue 全件表示対応の重なり回避アルゴリズム** + パン可能なスタジオ領域で対応
- 2D スプライトアセット調達 → LimeZu Modern Office + Modern Interiors で調達済み（docs/pixelart.md 参照）
- GitHub API レートリミット → ポーリング間隔の適切な設定 + キャッシュで対応
