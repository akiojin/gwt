### 技術コンテキスト

- Unity 6 + VContainer（DI）+ UniTask（非同期）
- シングルウィンドウ内で Additive Scene のアンロード→ロードによるスタジオ切替
- 現行 Rust/Tauri 版のマルチウィンドウ方式からの再設計
- PTYプロセスはシーンライフサイクルとは独立して管理

### 実装アプローチ

1. `IMultiProjectService` インターフェースを設計する
2. プロジェクトコンテキスト（PTY プロセス管理）の設計・実装（全PTY維持、制限なし）
3. Additive Scene のアンロード→ロードの切替メカニズムを実装する
4. スタジオ状態の永続化・復元を実装する
5. `Cmd+`` ホットキーとプロジェクト切替 UI を実装する
6. フェード/トランジション演出を実装する
7. VContainer DI 登録・テスト

### フェーズ分割

```
Phase S: セットアップ
  └─ IMultiProjectService インターフェース定義、データ型定義

Phase F: 基盤
  └─ プロジェクトコンテキスト管理（複数PTY同時管理）
  └─ Additive Scene アンロード→ロード切替メカニズム

Phase U: ユーザーストーリー実装
  └─ スタジオ状態永続化・復元
  └─ Cmd+` ホットキー・切替メニュー UI
  └─ フェード/トランジション演出
  └─ バックグラウンド PTY ライフサイクル管理

Phase FIN: 最終化
  └─ VContainer DI 登録、統合テスト
```
