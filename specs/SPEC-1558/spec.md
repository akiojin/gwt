### 背景

gwt は複数プロジェクト（Git リポジトリ）を同時に管理し、`Cmd+`` でプロジェクト間を切り替える機能を提供する。Unity 版ではシングルウィンドウ内で、1 スタジオ = 1 プロジェクトの対応で、スタジオ間の切替をフェード/トランジション演出で実現する。

現行 Rust/Tauri 版はマルチウィンドウでプロジェクトを管理しているが、Unity 版はシングルウィンドウのため、Additive Scene のアンロード→ロードによるスタジオ切替に再設計する。

**重要な制約**: プロジェクト切替時に PTY プロセス（エージェントセッション）を維持する。スタジオ（Unity シーン）はアンロード→ロードされるが、バックグラウンドの PTY プロセスは継続動作する。

### バックグラウンドPTY管理

- **全PTY維持（制限なし）**: 非アクティブプロジェクトのPTYプロセスも一時停止なく稼働継続する
- PTYプロセス数に上限は設けない
- 非アクティブプロジェクトのAgent（Developer）は切替先でも作業を継続する
- メモリ管理: 非アクティブプロジェクトのスタジオシーンはアンロードされるが、PTY管理サービスは維持

### ユーザーシナリオ

- **US-1** [P0]: `Cmd+`` でプロジェクト（スタジオ）切替メニューが表示され、別プロジェクトに切り替えられる
  - テスト: `Cmd+`` 押下で切替メニューが表示されること
  - テスト: プロジェクト選択で切替が実行されること

#### US-1 詳細

1. ユーザーが `Cmd+`` を押す
2. プロジェクト一覧がオーバーレイで表示される（最近開いたプロジェクト順）
3. ユーザーがプロジェクトを選択する
4. 現在のスタジオシーンが Additive Scene としてアンロードされる
5. 選択したプロジェクトのスタジオシーンが Additive Scene としてロードされる (`SceneManager.LoadSceneAsync(sceneName, LoadSceneMode.Additive)`)
6. フェード/トランジション演出で切替が完了する

- **US-2** [P1]: 切替時にフェード/トランジション演出が表示される
  - テスト: フェード演出が0.5秒以内で完了すること
- **US-3** [P0]: 切替後もエージェントセッションが維持され、作業が継続している
  - テスト: 切替後にエージェント PTY プロセスが継続動作していること
  - テスト: 切り戻し時にAgent状態がリアルタイムに反映されること

#### US-3 詳細

1. プロジェクト A でエージェント 3 体が作業中
2. ユーザーが `Cmd+`` でプロジェクト B に切替
3. プロジェクト A のエージェント PTY プロセスはバックグラウンドで**一時停止なく継続動作**
4. ユーザーが `Cmd+`` でプロジェクト A に戻る
5. エージェント 3 体の作業状態が正確に表示される（ターミナル出力も維持）

- **US-4** [P0]: 切替先のスタジオが前回の状態（デスク配置、Issue マーカー等）で復元される
  - テスト: デスク配置が復元されること
  - テスト: Issue マーカーが復元されること
- **US-5** [P1]: 最近開いたプロジェクトの一覧からクイック切替できる
  - テスト: 最近開いたプロジェクト一覧が表示されること
  - テスト: 一覧からクイック切替できること

### 機能要件

| ID | 要件 |
|----|------|
| FR-001 | `Cmd+`` ホットキーでプロジェクト切替メニューを表示する |
| FR-002 | プロジェクト切替時に Additive Scene をアンロード→ロードする (`SceneManager.LoadSceneAsync(sceneName, LoadSceneMode.Additive)`) |
| FR-003 | プロジェクト切替時に全PTYプロセス（エージェントセッション）を一時停止なく維持する |
| FR-004 | 切替時にフェード/トランジション演出を表示する |
| FR-005 | スタジオ状態（デスク配置、Issue マーカー、エージェント状態）を切替前に永続化し、切替後に復元する |
| FR-006 | 最近開いたプロジェクト一覧によるクイック切替をサポートする |
| FR-007 | 同時に複数プロジェクトの全PTYプロセスを制限なく管理する |
| FR-008 | VContainer で `IMultiProjectService` として DI 登録する |

### 非機能要件

| ID | 要件 |
|----|------|
| NFR-001 | プロジェクト切替は 3 秒以内に完了する（シーンロード含む） |
| NFR-002 | バックグラウンドの PTY プロセスがメモリリークしない |
| NFR-003 | 同時管理プロジェクト数に上限を設けない（メモリ制約の範囲内）。PTYプロセス数にも上限を設けない |
| NFR-004 | スタジオ切替時のトランジション演出（フェード）は 0.5秒以内 |

### インタビュー確定事項（2026-03-10追記）

**バックグラウンド継続:**
- プロジェクト切替時、非アクティブプロジェクトのPTYプロセスはバックグラウンドで継続動作
- Agent（Developer）は切替先でも作業を継続する
- 切り戻し時にAgent状態（running/idle/waiting_input等）がリアルタイムに反映される
- メモリ管理: 非アクティブプロジェクトのスタジオシーンはアンロードされるが、PTY管理サービスは維持
