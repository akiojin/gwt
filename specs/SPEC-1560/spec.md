### 背景

gwt Unity 6 版では、**フルRPG風サウンド**により、開発スタジオの雰囲気を演出する。カイロソフト「ゲーム発展途上国」風のレトロチップチューン（8bit/16bit）を基調としつつ、フィールドBGM、戦闘BGM（CI失敗）、勝利ジングル（PRマージ）、レベルアップジングル等のRPG的サウンド演出を導入する。BGM、効果音、UI 音の3層で構成し、Lead の TTS 音声との共存も考慮する。

Phase 5 での実装を予定するが、サウンドシステムの基盤（AudioManager、音量設定）は Phase 1 で準備する。

### サウンド方向性

- **ジャンル**: フルRPG風レトロチップチューン（8bit/16bit カイロソフト風）
- **参考**: カイロソフト「ゲーム発展途上国」「開幕!!パドックGP」のBGM + RPGサウンド演出
- **雰囲気**: 明るくポジティブな基本トーン、状況に応じてRPG的に変化
- **RPG的マッピング**:
  - フィールドBGM = 通常作業時BGM
  - 戦闘BGM = CI失敗時BGM
  - 勝利ジングル = PRマージ成功時
  - レベルアップジングル = スタジオレベルアップ時（#1555連動）

### サウンドアセット調達（2026-03-10インタビュー確定）

- **AI生成**: Suno、Udio等のAI音楽生成サービスを使用
- **ライセンス確認必須**: 商用利用可能なライセンスであることを確認してから使用
- AI生成により、レトロチップチューン＋RPG的サウンドを低コストで調達

### デフォルト設定

- **BGM: デフォルト無音（OFF）** — 開発ツールとしての使用時、BGMは集中の妨げになる可能性があるため
- **SE（効果音 + UI音）: デフォルトON** — 操作フィードバックとして有用なため
- ユーザーが Settings 画面で BGM を有効化するまで無音で起動
- SE は初回起動時から有効（ボタンクリック音、通知音等）
- BGM を好むユーザーは Settings から簡単に有効化可能

  - テスト: 通常フィールドBGMが再生されること
  - テスト: CI 全 Green 検知時に BGM バリエーションが切り替わること
  - テスト: CI 失敗時に戦闘BGMに切り替わること

  - テスト: エージェント雇用操作時に効果音が再生されること
  - テスト: デスク出現アニメーションと効果音のタイミングが同期すること
- **US-3** [P2]: UI 操作（ボタンクリック、パネル開閉）で適切な UI 音が再生される

  - テスト: パネル開閉時に UI 音が再生されること

### 機能要件

#### BGM

| ID | 要件 |
|----|------|
| FR-001 | スタジオ フィールドBGM を再生する（通常作業時） |
| FR-002 | CI 全 Green 時に明るい BGM バリエーションに切り替える |
| FR-003 | CI 失敗時に戦闘BGM（暗い/緊迫した BGM）に切り替える |
| FR-004 | BGM の切り替えはクロスフェード（1秒）で滑らかに遷移する |
| FR-005 | BGM はループ再生する |

#### 効果音

| ID | 要件 |
|----|------|
| FR-006 | デスク出現時に効果音を再生する |
| FR-007 | Issue マーカー出現時に効果音を再生する |
| FR-008 | エージェント雇用時に効果音を再生する（ポジティブ音） |
| FR-009 | エージェント解雇時に効果音を再生する |
| FR-010 | 通知音を再生する（新 Issue、PR 作成、CI 結果等） |
| FR-011 | エラー音を再生する（エージェントエラー、操作失敗等） |
| FR-012 | レベルアップジングルを再生する（Phase 5 ゲーミフィケーション連動、#1555） |

#### ジングル（RPG演出）

| ID | 要件 |
|----|------|
| FR-023 | PRマージ成功時に勝利ジングルを再生する |
| FR-024 | レベルアップ時にレベルアップジングルを再生する（#1555連動） |
| FR-025 | CI失敗→復旧時に戦闘勝利ジングルを再生する |

#### UI 音

| ID | 要件 |
|----|------|
| FR-013 | ボタンクリック時に UI 音を再生する |
| FR-014 | パネル開閉時に UI 音を再生する |
| FR-015 | 設定変更時に確認音を再生する |
| FR-016 | メニュー遷移時に遷移音を再生する |

#### サウンドシステム基盤

| ID | 要件 |
|----|------|
| FR-017 | AudioManager を VContainer で Singleton として DI 登録する |
| FR-018 | BGM はデフォルト無音（OFF）、SE（効果音 + UI音）はデフォルト ON。ユーザーが Settings 画面で個別に切替可能 |
| FR-019 | 各チャンネルの音量設定を永続化する（~/.gwt/ 内 JSON） |
| FR-020 | マスター音量 + チャンネル別音量の2段階制御をサポートする |
| FR-021 | Lead TTS 再生中に BGM 音量を自動で下げる（ダッキング、-12dB 程度） |
| FR-022 | TTS 再生終了後に BGM 音量を元に戻す（フェードイン 0.5秒） |

### 非機能要件

| ID | 要件 |
|----|------|
| NFR-001 | サウンドファイル形式: .ogg (BGM), .wav (効果音/UI音) |
| NFR-002 | BGM ファイルサイズ: 1曲あたり 2MB 以下（ループ対応） |
| NFR-003 | 効果音の遅延: トリガーから再生開始まで 50ms 以内 |
| NFR-004 | 同時再生: BGM 1 + 効果音 8 + UI 音 4 の同時再生をサポートする |
| NFR-005 | AudioSource プーリング: 効果音用 AudioSource を ObjectPool で管理する |
| NFR-006 | メモリ使用量: 全サウンドアセットの常駐メモリは 50MB 以下 |
| NFR-007 | クロスフェード中に音割れ・ノイズが発生しないこと |
| NFR-008 | サウンドアセットはAI生成（Suno/Udio等）で調達する。商用利用可能なライセンスであることを確認必須 |
