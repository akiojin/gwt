# ヘルプオーバーレイ

## Background

gwt-tui に Ctrl+G, ? でアクセスできるヘルプオーバーレイを追加する。全キーバインド一覧と、現在のコンテキスト（メイン画面 / 管理画面）に応じた操作説明を表示する。ESC で閉じる。

## User Stories

### US-1: キーバインド一覧を確認する

開発者として、Ctrl+G, ? を押して全キーバインドの一覧を確認し、操作方法を素早く把握したい。

### US-2: コンテキストに応じた操作説明を見る

開発者として、現在の画面（メイン画面 or 管理画面）に応じた操作説明を見て、今使える操作を把握したい。

### US-3: オーバーレイを素早く閉じる

開発者として、ESC を押してヘルプオーバーレイを即座に閉じ、作業に戻りたい。

## Acceptance Scenarios

### AS-1: ヘルプオーバーレイの表示

- gwt-tui で Ctrl+G, ? を押す
- フルスクリーンのオーバーレイが表示される
- 全キーバインドがカテゴリ別に一覧表示される

### AS-2: コンテキスト別表示

- メイン画面で Ctrl+G, ? を押す
- メイン画面のキーバインド（Shell/Agent タブ操作、PTY 入力等）がハイライトされる
- 管理画面で Ctrl+G, ? を押す
- 管理画面のキーバインド（タブ切り替え、検索等）がハイライトされる

### AS-3: オーバーレイの閉じ

- ヘルプオーバーレイが表示されている状態
- ESC を押す
- オーバーレイが閉じ、元の画面に戻る

### AS-4: スクロール対応

- キーバインドが画面に収まらない場合
- 上下スクロールで全項目を確認できる

## Functional Requirements

| ID | 要件 |
|----|------|
| FR-001 | Ctrl+G, ? でヘルプオーバーレイを表示する |
| FR-002 | 全キーバインドをカテゴリ別（General / Main Screen / Management Screen / Shell / Navigation）に一覧表示する |
| FR-003 | 現在のコンテキスト（メイン画面 / 管理画面）に応じてキーバインドをハイライト表示する |
| FR-004 | ESC でオーバーレイを閉じる |
| FR-005 | コンテンツが画面に収まらない場合はスクロールをサポートする |
| FR-006 | キーバインド定義をコードから自動収集する（ハードコードしない） |

## Non-Functional Requirements

| ID | 要件 |
|----|------|
| NFR-001 | オーバーレイの表示は 50ms 以内に完了する |
| NFR-002 | オーバーレイ表示中もバックグラウンドの PTY 出力バッファリングは継続する |

## Success Criteria

| ID | 基準 |
|----|------|
| SC-001 | Ctrl+G, ? でヘルプオーバーレイが表示される |
| SC-002 | 全キーバインドがカテゴリ別に正しく表示される |
| SC-003 | コンテキストに応じたハイライトが機能する |
| SC-004 | ESC でオーバーレイが閉じる |
| SC-005 | スクロールで全項目を確認できる |
