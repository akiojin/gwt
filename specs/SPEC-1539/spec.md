### 背景

gwt (Git Worktree Manager) を Rust/Tauri/Svelte ベースのデスクトップアプリから Unity 6 ベースのトップダウン ¾ ビューゲーム型アプリケーションへ全面移行する。
現在の gwt は **132 の Tauri コマンド**（28モジュール）、33の Svelte コンポーネント、xterm.js ターミナル統合を持つ大規模アプリであり、これを Unity (C#) で完全に再実装する。

### ビジョン

- **開発スタジオメタファー**: カイロソフト「ゲーム発展途上国（Game Dev Story）」に着想を得たトップダウン ¾ ビュー視点の開発スタジオ内で、Git操作・エージェント管理・ターミナル操作を行う
- **スタジオ内オブジェクトのマッピング**:
  - デスク/PC = Worktree（各ブランチの作業領域）
  - 開発者キャラクター（2Dスプライト） = Agent（Claude Code 等）
  - 浮遊マーカー（!マーク） = Issue（バグ/課題）
  - PR バッジ = Pull Request（デスクに付随）
- **Lead 入力は常時表示 HUD**: RTS風コンソールとして画面上に常に表示される入力フィールド（ミーティングルームの概念は不採用）
- **Lead キャラクター**: Lead にはキャラクター性があり、音声会話対応（ローカル Qwen3-ASR + ローカル TTS）
- **Lead→Agent 協調は常時アクティブ**: スタジオを開いた時点でエージェント協調が有効。単一エージェント起動・手動ターミナルも可能
- **動的スタジオ拡張**: Issue マーカー位置にデスクが出現、Agent 追加 → キャラクターが出現
- **雰囲気変化**: CI 失敗 → デスク単位で暗く/赤くなる、全てグリーン → 明るい雰囲気に変化（URP の2Dライティングエフェクトで実現）
- **シングルウィンドウ完結**: ゲームらしく全操作を1つのウィンドウ内で完結させる

### ビジュアルデザイン

- **トップダウン ¾ ビュー**: カイロソフト「ゲーム発展途上国」スタイル（3Dではなくトップダウン ¾ ビュー）
- **2D ピクセルアートスプライト**: 3Dモデルは不採用。ピクセルアートスタイルのスプライトアニメーション:
  - `running` → PC でタイピング中
  - `waiting_input` → 手を振っている（入力待ち）
  - `stopped` → 休憩中（デスクから離れている）
  - `idle` → デスクに座って待機
- **URP は2Dライティング用に維持**: 雰囲気変化（CI状態に応じた照明変更等）のために URP を使用
- **アセット**: LimeZu エコシステム + Unity Asset Store（docs/pixelart.md 参照）

### 廃止される技術

| 項目 | 理由 |
|------|------|
| Rust (gwt-core, gwt-tauri) | C# で全機能再実装 |
| Svelte 5 (gwt-gui) | Unity UI に置換 |
| Tauri v2 | Unity スタンドアロンに置換 |
| xterm.js | 自前 TerminalEmulator + TextMeshPro ベースターミナルに置換 |
| XtermSharp | 不採用（自前 TerminalEmulator を実装） |
| gwt プラグイン | 完全廃止 |
| 「Project Mode」名称 | 機能は維持するが名称を廃止。Lead→Agent 協調は常時アクティブ |
| 掲示板 (Bulletin Board) | 浮遊 Issue マーカーに置換 |
| モード切替 UI | すべてスタジオ内で統合 |
| TOML 設定ファイル | JSON に完全統一。agents.toml 互換も廃止、Tomlyn 依存不要 |
| Sparkle / WinSparkle | GitHub Release API 自前実装に置換 |

### 廃止されるコマンド（シングルウィンドウ化・UI 再設計により不要）

以下の Rust 版コマンドは Unity 版の設計変更（シングルウィンドウ化、タブシステム廃止）により再実装対象外とする:

| コマンド | 廃止理由 |
|---------|---------|
| `get_current_window_label` | シングルウィンドウ化によりウィンドウラベル不要 |
| `try_acquire_window_restore_leader` | マルチウィンドウのリーダー選出不要 |
| `release_window_restore_leader` | 同上 |
| `open_gwt_window` | シングルウィンドウのため新規ウィンドウ不要 |
| `sync_window_agent_tabs` | タブシステム廃止→エンティティクリックオーバーレイに置換 |
| `sync_window_active_tab` | 同上 |
| `greet` | 開発/テスト用コマンド |

### ユーザーシナリオ

- **US-1 [P0]**: Unity スタンドアロンアプリとして macOS/Windows/Linux で起動し、トップダウン ¾ ビューのスタジオ画面が表示される
- **US-2 [P0]**: スタジオ内でエンティティ（デスク=Worktree、キャラ=Agent、マーカー=Issue）をクリックして詳細操作ができる
- **US-3 [P0]**: Lead→Agent 協調が常時アクティブで、単一エージェント起動・手動ターミナルも選択可能
- **US-4 [P0]**: 全132コマンド中125コマンドの機能が Unity 版で利用可能
- **US-5 [P1]**: ~/.gwt/ のデータが Rust 版から自動マイグレーションされ、既存環境をシームレスに引き継げる
- **US-6 [P1]**: CI 状態に応じてスタジオの雰囲気（ライティング）が動的に変化する
