# 機能仕様: タブ有効化時チラつきとタブ切替カクつきの改善

## 背景
- Windows を中心に、タブ切替時およびタブ有効化直後に画面のチラつきが発生する。
- 非アクティブターミナルを含む再描画・リサイズ処理が過剰に走り、全体操作がカクつく。

## ユーザーストーリー
### ユーザーストーリー 1 - タブ有効化時のチラつきを抑えたい (優先度: P0)
ユーザーとして、ターミナルタブをアクティブにした瞬間に描画が崩れたり白く点滅したりせず、安定した表示で切り替わってほしい。

**独立したテスト**: ターミナルタブ切替時、fit 完了前の中間描画を見せずに表示されることを確認する。

### ユーザーストーリー 2 - 全体のタブ切替を滑らかにしたい (優先度: P0)
ユーザーとして、タブ種別に依存せず、切替時のカクつきを抑えて快適に操作したい。

**独立したテスト**: Terminal↔Terminal / Terminal↔非Terminal で操作体感の劣化がないことを確認する。

## 機能要件
- **FR-001**: システムは、ターミナルタブが有効化されたとき、`fit` と `resize_terminal` の完了後に表示を有効化しなければならない。
- **FR-002**: システムは、ready 通知が得られない場合でも短時間でフォールバック表示し、無表示固定を防止しなければならない。
- **FR-003**: システムは、非アクティブターミナルに対して `ResizeObserver` 起因の `fit` / `resize_terminal` を実行してはならない。
- **FR-004**: システムは、`resize_terminal` 呼び出しで同一行列サイズの重複通知を抑止しなければならない。
- **FR-005**: システムは、非ターミナルタブコンテンツを Keep-Alive し、タブ切替時の再マウントを抑止しなければならない。
- **FR-006**: システムは、既存のコピー/ペースト、キーボードショートカット、タブD&D、タブClose挙動を維持しなければならない。

## 非機能要件
- **NFR-001**: 改修は全OS共通の挙動として適用し、OS分岐を増やさないこと。
- **NFR-002**: 既存UIアニメーション仕様は変更しないこと。

## 成功基準
- **SC-001**: ターミナル有効化直後の目視チラつきがほぼ解消される。
- **SC-002**: タブ切替での体感カクつきが改善される。
- **SC-003**: 既存ユニットテストに加え、追加したユニットテスト/E2E性能チェックが通過する。
