# 機能仕様: URLクリック時の外部ブラウザ起動統一

**仕様ID**: `SPEC-d95a7e0c`
**作成日**: 2026-02-20
**更新日**: 2026-02-20
**ステータス**: ドラフト
**カテゴリ**: GUI
**依存仕様**:

- なし

**入力**: ユーザー説明: "gwtに表示されるURLをクリックした場合に、外部ブラウザで開く"

## 背景

- 現状、画面によってリンククリック時の挙動が統一されておらず、`window.open` と `plugin-shell` が混在している。
- Terminal 上の URL クリックも含め、ユーザーは常に OS 既定の外部ブラウザで開くことを期待している。

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - URLを外部ブラウザで開く (優先度: P0)

ユーザーとして、gwt上で表示されたURLをクリックしたときに、常に外部ブラウザで開きたい。

**独立したテスト**: Terminal / Markdown / Issue / PR / Workflow で表示された `https://` リンクをクリックし、外部起動処理が呼ばれることを確認する。

**受け入れシナリオ**:

1. **前提条件** gwt の Terminal に `https://example.com` が表示されている、**操作** URL をクリックする、**期待結果** OS既定ブラウザで URL が開く。
2. **前提条件** gwt の Issue/PR/Markdown などに `https://` リンクが表示されている、**操作** リンクをクリックする、**期待結果** OS既定ブラウザで URL が開く。

---

### ユーザーストーリー 2 - 安全なスキーム制限 (優先度: P1)

ユーザーとして、安全でないURLスキームは誤って外部起動されたくない。

**独立したテスト**: `javascript:` / `mailto:` など `http/https` 以外のリンク文字列を与え、外部起動処理が呼ばれないことを確認する。

**受け入れシナリオ**:

1. **前提条件** クリック対象が `javascript:alert(1)`、**操作** クリック処理を実行する、**期待結果** 外部起動されない。
2. **前提条件** クリック対象が `mailto:test@example.com`、**操作** クリック処理を実行する、**期待結果** 外部起動されない。

## エッジケース

- `plugin-shell` 呼び出しが失敗した場合でも、UI がクラッシュしないこと。
- 不正URL（パース不可文字列）を受け取っても例外で停止しないこと。
- 修飾キー付きクリック（Cmd/Ctrl+Click）や右クリックを通常挙動のまま阻害しないこと。

## 要件 *(必須)*

### 機能要件

- **FR-001**: gwt内のクリック可能な `http/https` URL は、画面種別（Terminal/Markdown/Issue/PR/Workflow）に依存せず外部ブラウザ起動処理に統一しなければならない。
- **FR-002**: Terminal の URL クリックは xterm のリンクハンドラで受け取り、外部ブラウザ起動処理を呼び出さなければならない。
- **FR-003**: UIの通常リンクはグローバルなクリック委譲で捕捉し、`http/https` のみを外部起動しなければならない。
- **FR-004**: 外部起動対象スキームは `http` / `https` のみとし、それ以外は起動してはならない。
- **FR-005**: Tauri ランタイムでは `@tauri-apps/plugin-shell` の `open` を優先し、失敗時もアプリの主要機能を継続させなければならない。
- **FR-006**: Tauri 側で shell plugin 初期化と必要権限（`shell:allow-open`）を設定しなければならない。
- **FR-007**: 本仕様の受け入れ条件を満たす自動テスト（ユニット/コンポーネント）を追加しなければならない。

### 非機能要件

- **NFR-001**: 既存機能（リンク以外のクリックやタブ操作）へ回帰を起こさないこと。
- **NFR-002**: 変更は既存構造を優先し、重複するリンク起動ロジックを増やさないこと。

## 制約と仮定

- 本仕様で許可する URL スキームは `http/https` のみとする。
- 既存の UI 文言やレイアウトは変更しない。
- ブラウザ起動可否の最終判断は OS と Tauri plugin 側の制御に委ねる。

## 成功基準 *(必須)*

- **SC-001**: `gwt-gui` テストで、Terminal と UI リンクの外部起動ハンドリングに関する追加ケースがすべて成功する。
- **SC-002**: `http/https` 以外のスキームが外部起動されないことをテストで確認できる。
- **SC-003**: Tauri capability 設定に `shell:allow-open` が追加され、起動権限不足による実行時エラーが発生しない。
