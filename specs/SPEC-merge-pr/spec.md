# 機能仕様: Worktree詳細ビューでMergeableクリックによるマージ実行

**仕様ID**: `SPEC-merge-pr`
**作成日**: 2026-02-26
**更新日**: 2026-02-26
**ステータス**: ドラフト
**カテゴリ**: GUI
**依存仕様**: なし

**入力**: ユーザー説明: "Worktree詳細ビューのPRタブにあるMergeableバッジをクリックしてPRマージを実行できるようにする"

## 背景

- PRタブのMergeableバッジは静的な`<span>`で情報表示のみ
- ユーザーがGitHub画面に遷移せずにGWT GUI内でマージを完結させたい
- バックエンドにPRマージ用Tauriコマンドが存在しない

## ユーザーシナリオとテスト

### ユーザーストーリー 1 - Mergeableバッジクリックでマージ実行 (優先度: P0)

ユーザーとして、OPENかつMERGEABLEなPRのMergeableバッジをクリックして確認モーダルからマージを実行したい。

**独立したテスト**: Mergeableバッジがボタン化され、クリックで確認モーダルが表示される

**受け入れシナリオ**:

1. **前提条件** state=OPEN, mergeable=MERGEABLE のPRが表示されている、**操作** Mergeableバッジをクリック、**期待結果** 確認モーダルが表示される
2. **前提条件** 確認モーダルが表示されている、**操作** Mergeボタンをクリック、**期待結果** APIが呼び出され "Merging..." 表示後、成功トーストが表示される
3. **前提条件** マージ成功後、**操作** なし（自動）、**期待結果** ポーリングでPR詳細がリフレッシュされ state=MERGED バッジに変化する

---

### ユーザーストーリー 2 - 確認モーダルの情報表示 (優先度: P0)

ユーザーとして、マージ前にPR番号・タイトル・ブランチ名・CI状態・レビュー状態を確認したい。

**独立したテスト**: モーダルにPR情報が正しく表示される

**受け入れシナリオ**:

1. **前提条件** 確認モーダルが表示されている、**操作** なし、**期待結果** PR番号・タイトル・ブランチ名 (head -> base) が表示される
2. **前提条件** 確認モーダルが表示されている、**操作** なし、**期待結果** CIチェック結果とレビュー承認状態がアイコンで表示される

---

### ユーザーストーリー 3 - マージ不可PRの表示 (優先度: P1)

ユーザーとして、マージできないPRでは誤操作を防ぐためバッジがクリック不可であることを確認したい。

**独立したテスト**: CLOSED/CONFLICTING/UNKNOWN/MERGEDのPRではボタン化されない

**受け入れシナリオ**:

1. **前提条件** state=CLOSED のPR、**操作** バッジ確認、**期待結果** `<span>` 要素のまま（クリック不可）
2. **前提条件** mergeable=CONFLICTING のPR、**操作** バッジ確認、**期待結果** `<span>` 要素のまま
3. **前提条件** state=MERGED のPR、**操作** バッジ確認、**期待結果** `<span>` 要素のまま

---

### ユーザーストーリー 4 - マージ失敗時のエラーハンドリング (優先度: P1)

ユーザーとして、マージAPIが失敗した場合にエラー内容を確認し、再試行できるようにしたい。

**独立したテスト**: API失敗時にエラートーストが表示され、再度バッジクリックで再試行可能

**受け入れシナリオ**:

1. **前提条件** マージAPI実行中、**操作** APIがエラーを返す、**期待結果** モーダルが自動クローズし、エラートーストが表示される
2. **前提条件** エラー発生後、**操作** Mergeableバッジを再クリック、**期待結果** 確認モーダルが再表示され、再試行が可能

## エッジケース

- CLOSED PRではマージボタン非表示
- mergeStateStatus=BEHIND でも mergeable=MERGEABLE ならクリック可能
- API タイムアウト (15秒) 時はエラートースト表示
- マージ中にユーザーがブランチを切り替えた場合、ポーリングは中断される

## 要件

### 機能要件

- **FR-001**: state=OPEN かつ mergeable=MERGEABLE の場合、Mergeableバッジを `<button>` 化する
- **FR-002**: ボタンクリックで確認モーダル (MergeConfirmModal) を表示する
- **FR-003**: 確認モーダルにPR番号・タイトル・ブランチ名・CI状態・レビュー状態を表示する
- **FR-004**: モーダルのMergeボタンで `gh api PUT /repos/{owner}/{repo}/pulls/{pr_number}/merge` を呼び出す
- **FR-005**: マージ中は Merge ボタンを disabled + "Merging..." テキスト表示
- **FR-006**: マージ成功時に toastBus 経由でトースト通知を表示する
- **FR-007**: マージ成功後、ポーリングで PR 詳細をリフレッシュ (state=MERGED まで)
- **FR-008**: マージ失敗時、モーダル自動クローズ + エラートースト表示

### 非機能要件

- **NFR-001**: マージAPIタイムアウトは15秒

## 制約と仮定

- GitHub REST API (`gh api`) を使用しマージを実行する
- マージ方式はリポジトリのデフォルト (merge_method 省略)
- toastBus は errorBus と同様のグローバルイベントバスパターンで新設
