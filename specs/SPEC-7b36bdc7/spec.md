# 機能仕様: 設定画面で UI / Terminal フォントファミリーを選択可能にする

**仕様ID**: `SPEC-7b36bdc7`
**作成日**: 2026-02-20
**更新日**: 2026-02-20
**ステータス**: 実装済み
**カテゴリ**: GUI
**依存仕様**:

- `SPEC-5cd3bfec`（設定画面タブ構成）
- `SPEC-f490dded`（ターミナルタブ基盤）

**入力**: ユーザー説明: "フォントを変更できるようにしたい。設定画面で選択可能"

## 背景

- 既存の設定画面では `UI Font Size` / `Terminal Font Size` のサイズ指定は可能だが、フォントファミリーの選択ができない
- UI とターミナルの視認性要件は異なるため、サイズだけでなくフォントファミリーの個別設定が必要
- 設定保存前に見た目を確認できるプレビューと、保存しない場合のロールバックが必要

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - Appearance タブでフォントファミリーを即時プレビューする (優先度: P0)

ユーザーとして、Settings の Appearance タブで UI / Terminal のフォントファミリーを選択し、保存前に即時プレビューしたい。

**独立したテスト**: Settings を開いてフォントセレクトを変更した際、CSS 変数およびターミナル側イベント反映が即時に起きることを確認する。

**受け入れシナリオ**:

1. **前提条件** Settings の Appearance タブを表示、**操作** `UI Font Family` を別プリセットへ変更、**期待結果** `--ui-font-family` が即時更新される
2. **前提条件** Settings の Appearance タブを表示、**操作** `Terminal Font Family` を別プリセットへ変更、**期待結果** `--terminal-font-family` と `gwt-terminal-font-family` イベントが即時反映される

---

### ユーザーストーリー 2 - 保存したフォントファミリーが再起動後も維持される (優先度: P0)

ユーザーとして、選択したフォントファミリーを保存したら、次回起動時にも同じフォントで表示されてほしい。

**独立したテスト**: `save_settings` に `ui_font_family` / `terminal_font_family` が含まれ、起動時 `get_settings` の値で初期適用されることを確認する。

**受け入れシナリオ**:

1. **前提条件** Settings で両フォントを変更済み、**操作** Save を押下、**期待結果** 設定保存ペイロードに両フォントキーが含まれる
2. **前提条件** フォント設定保存済み、**操作** アプリ起動、**期待結果** 初回描画前に UI/Terminal のフォントが保存値で適用される

---

### ユーザーストーリー 3 - 保存せず閉じた場合は元に戻る (優先度: P1)

ユーザーとして、フォントを試しても保存せず Close した場合、保存済みの設定値に戻ってほしい。

**独立したテスト**: フォント変更後に Close し、`--ui-font-family` / `--terminal-font-family` / ターミナル側保持値が保存済み値へ復帰することを確認する。

**受け入れシナリオ**:

1. **前提条件** Settings 表示中にフォントを未保存変更済み、**操作** Close を押下、**期待結果** 両フォントが保存済み値へロールバックされる

## エッジケース

- 設定ファイルに空文字や未知のフォント値がある場合は、既定プリセットへフォールバックする
- `get_settings` がフォント項目を返さない互換ケースでも、起動と設定画面は既定値で継続動作する
- 保存失敗時はプレビュー状態を維持しつつ、エラーメッセージを表示する

## 要件 *(必須)*

### 機能要件

- **FR-001**: 設定データモデルに `appearance.ui_font_family` と `appearance.terminal_font_family` を追加し、既定値を持つこと
- **FR-002**: Settings の Appearance タブに `UI Font Family` と `Terminal Font Family` のプリセット選択 UI を表示すること
- **FR-003**: フォント選択変更時、保存前でも UI とターミナルプレビューを即時反映すること
- **FR-004**: Save 時、`ui_font_family` / `terminal_font_family` を `save_settings` の保存対象に含めること
- **FR-005**: Close 時、未保存のフォント変更を破棄し、保存済み値へ復元すること
- **FR-006**: 起動時 `get_settings` 結果からフォントファミリーを読み取り、マウント前に適用すること
- **FR-007**: フォント値の正規化を行い、空値・未知値は既定フォントへフォールバックすること
- **FR-008**: ユニットテストと E2E テストで保存・即時反映・Close ロールバックを検証すること

### 非機能要件

- **NFR-001**: フォント反映はアプリ再起動不要で完結すること
- **NFR-002**: フォント反映処理は既存設定UIの操作感を劣化させないこと（体感で即時）

## 制約と仮定

- フォント選択は自由入力ではなく、定義済みプリセットのみを対象とする
- 保存値は CSS `font-family` スタック文字列として扱う
- 既存の `ui_font_size` / `terminal_font_size` 挙動は維持する

## 成功基準 *(必須)*

- **SC-001**: Appearance タブで UI / Terminal のフォントセレクトが表示され、選択操作できる
- **SC-002**: Save 時に `ui_font_family` / `terminal_font_family` が保存ペイロードに含まれる
- **SC-003**: Close without save でフォントプレビューが保存済み値に戻る
- **SC-004**: 起動時に保存済みフォントが初期適用される
- **SC-005**: 関連ユニットテストと E2E テストが GREEN になる
