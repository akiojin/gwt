### 背景

トップダウン ¾ ビュー開発スタジオの上に重ねる UI レイヤーとして、RTS 風の HUD 要素とゲームスタイルのメニューを組み合わせたシステムを構築する。通知はスタジオ内のビジュアルエフェクトと RTS 風コンソールウィンドウの両方で表現し、設定は ESC キーで開くゲームスタイルのオプションメニューからアクセスする。

UI素材は Pixel UI &amp; HUD (Dead Revolver, $4.99) を基盤とし、Kenney Pixel UI Pack (無料, CC0) で補完する。詳細は `docs/pixelart.md` を参照。

**シングルウィンドウ**: 全操作をシングルウィンドウ内で完結する。別ウィンドウへの遷移は一切行わない。

**Lead への指示は HUD の常時表示入力フィールドから行う。ミーティングルームの概念は存在せず、Lead はスタジオ内に常駐しており、HUD 経由で常時アクセス可能。**

**Lead 質問 UI**: Lead がユーザーに質問する際は、浮遊「?」マーカー + バルーン + 選択肢ボタンで表現する。World Space Canvas でスタジオ内に表示し、#1547 の Issue「!」マーカーと同様の視覚言語を使用する。

**キーバインド**: フォーカスベース切替方式。ターミナルフォーカス時はターミナルキーバインド優先、スタジオフォーカス時はアプリショートカット優先。

**Git操作 UI**: ターミナル経由で実行する。エージェントが実行するか、ユーザーがプレーンターミナルで直接実行する。ゲームライク Git UI は不要。

**ローカライゼーション**: Unity Localization パッケージ (Smart Strings + アセットテーブル) による英語 + 日本語対応。ESC メニューから言語切替が可能。**デフォルト言語は OS のシステム言語設定に追従する。**

**ターミナル**: ターミナルはスタジオビュー上にフローティングするオーバーレイパネルとして表示する。

**Git 詳細情報（diff、コミット履歴、stash）はオーバーレイパネルで表示する。**

**Issue マーカー UI**: スタジオ内の Issue マーカーをクリックすると、オーバーレイで Issue 詳細が表示され、エージェント雇用（worktree 作成）ボタンから直接作業を開始できる。

**半透明デスク UI**: 空きデスクをクリックすると worktree 作成 + エージェント雇用フローが開始される。

**worktree 作成 UI**: HUD ボタンからの作成と Lead への指示による作成の両方をサポートする。

**SPEC エディタパネル**: SPEC 生成時の専用エディタパネルはオーバーレイパネルの一種として実装する。レイアウトは左=チャット、右=マークダウンプレビューの2ペイン構成。既存のオーバーレイシステム（FR-005）に統合する。

**マークダウンレンダリング**: 完全 GitHub Flavored Markdown (GFM) を自前実装でサポートする。**Markdig 等の外部パーサーは使用せず、自前 GFM パーサー + TextMeshPro レンダラーで実装する（将来パッケージ化前提）。** テーブル、コードブロック（シンタックスハイライト）、画像、チェックボックス、リンク、リスト等すべてに対応し、GitHub の Issue/PR 本文の見た目を再現する。

**コンテキストメニュー描画**: Screen Space で描画する。

**エラー通知3段階**:

| 重要度 | 表示方式 |
|--------|---------|
| Error | トースト通知 + コンソールウィンドウ |
| Warning | コンソールウィンドウのみ |
| Info | ログのみ（コンソール非表示） |

**現行 gwt UI 要素からの移行マッピング:**

| 現行 UI 要素 | スタジオ UI での実装 |
|-------------|---------------------|
| StatusBar | HUD（ブランチ、エージェント数、ステータス） |
| Toast 通知 | スタジオ内エフェクト + コンソールウィンドウ |
| Settings パネル | ESC ゲームスタイル設定メニュー |
| Agent Launch フォーム | オーバーレイパネル or Lead 委任 |
| Error 表示 | コンソールウィンドウ + スタジオ内エフェクト |
| Sidebar（ブランチリスト） | スタジオ内デスク（Worktree）表現 |
| Tab システム | エンティティクリック → オーバーレイ詳細パネル |
| Lead への指示 | HUD 常時表示入力フィールド |
| Git diff / commits / stash | オーバーレイパネル（デスククリック時表示） |

### インタビュー確定事項（2026-03-10追記）

**HUD常時表示内容:**
- プロジェクト名
- Active Agent数（現在稼働中のAgent数）
- Open PR数
- Lead入力フィールド（既存仕様）

**通知の3層構造:**
| 層 | 用途 | 表示方式 |
|----|------|---------|
| トースト通知 | 即時通知（エラー、完了、警告） | 画面上部/下部に一時表示、5秒で自動消去 |
| ログリスト | 履歴参照（RTS風コンソール） | 画面右下のコンソールウィンドウ、スクロール可 |
| OS通知 | アプリ非フォーカス時の通知 | macOS/Windows/LinuxのOS標準通知 |

**エラー通知3段階:**
| 重要度 | 表示方式 |
|--------|---------|
| Error | トースト通知 + コンソールウィンドウ |
| Warning | コンソールウィンドウのみ |
| Info | ログのみ（コンソール非表示） |

**キーボードフォーカス管理:**
- ターミナルフォーカス時: ターミナルが全キー入力を飲み込む（アプリショートカット無効）
- フォーカス解除: パネル外クリックで解除（ESCではない）
- ESCキー: パネル/メニューを閉じる（フォーカス解除ではない）

**オンボーディング:**
- オンボーディングフロー（チュートリアル）は実装しない
- Lead自体がガイド役を果たす設計

**オフライン時:**
- ローカル機能（PTY、Gitローカル操作、ターミナル）のみ継続
- GitHub連携機能は一時停止（ポーリング停止）
- 再接続時に自動復帰

**Lead質問UI:**
- Lead がユーザーに質問する際: 浮遊「?」+ バルーン + 選択肢ボタン
- World Space Canvas でスタジオ内に表示
- #1547 の Issue「!」マーカーと同様の視覚言語

**SPECエディタパネル:**
- SPEC生成時の専用エディタパネルはオーバーレイパネルの一種として実装
- レイアウト: 左=チャット、右=マークダウンプレビュー
- 既存のオーバーレイシステム（FR-005）に統合

**マークダウンレンダリング:**
- 完全GitHub Flavored Markdown (GFM) サポート
- **自前完全実装（Markdigパーサー不使用）: 自前GFMパーサー + TextMeshProレンダラー**
- **将来パッケージ化前提の設計**
- テーブル、コードブロック（シンタックスハイライト）、画像、チェックボックス、リンク、リスト等すべて対応
- GitHubのIssue/PR本文の見た目を再現

**ローカライゼーション:**
- Unity Localization パッケージ (Smart Strings + アセットテーブル)
- 英語 + 日本語対応
- **デフォルト言語は OS のシステム言語設定に追従する**

**コンテキストメニュー描画:**
- Screen Space で描画する
